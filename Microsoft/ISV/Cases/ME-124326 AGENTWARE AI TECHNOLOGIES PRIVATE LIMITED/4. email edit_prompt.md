# Email Validation Edit Prompt

Case Path: `[case folder path]`

## Input
Read `1. Scope.md`, `2. after meeting.md`, and `3. email.md`.

## Goal
Validate `3. email.md` against the actual meeting record and write the result to `4. email edit.md`.

## Validation Rules
Use `3. email.md` as the base document. Keep the subject, wording, order, sections, tables, links, and closing unchanged unless the content is incorrect.

Use `1. Scope.md` only for company identity information: company name, country, case ID, contact names, and email addresses.

Do not use `1. Scope.md` for requirements, architecture, consultation needs, roadblocks, recommendations, or meeting conclusions.

Use `2. after meeting.md` as the source of truth for meeting topics, decisions, technical facts, and follow-up items.

## Edit Rules
Correct only unsupported or inaccurate sentences, sections, table rows, service names, decisions, dates, owners, or recommendations.

Leave all validated content exactly as written in `3. email.md`.

If no issue is found, copy `3. email.md` to `4. email edit.md` without changes.

## Output
Write only the final email content, including the subject. Do not include validation notes, explanations, change summaries, or prompt text.
