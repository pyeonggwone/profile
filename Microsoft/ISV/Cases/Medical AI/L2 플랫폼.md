이 다이어그램은 MedicalAI AiTiA ECO CENTER 플랫폼의 전체 서비스 구조를 나타내며,
병원 온프레미스 환경과 MedicalAI 클라우드 간의 연결 방식을 조감도 수준에서 보여준다.

[고객 환경 - Hospital A]
주요 고객인 Hospital A는 병원 내부에 FTP/SFTP 수신 채널을 두고,
Apache Flume 기반 데이터 수집 파이프라인을 운영한다.
수집된 의료 데이터는 Docker 컨테이너 환경에서 PII Masking 처리를 거쳐
개인식별정보가 제거된 후, EMR 및 PACS와 연결된 VM을 통해
MedicalAI 클라우드로 암호화 전송된다.
Azure Arc를 통해 이 VM에 대한 원격 SW 업데이트 및 헬스 모니터링이 수행된다.

[고객 환경 - Hospital B, C, D]
중소 규모 병원들은 경량 Kubernetes 배포판인 K3S 클러스터로 구성되어
엣지 환경에서 독립적으로 운영되며, MedicalAI 클라우드와 연결된다.

[MedicalAI 클라우드]
클라우드 진입점에는 Load Balancer가 위치하여 트래픽을 분산한다.
내부는 Azure VM과 AKS 조합으로 서비스를 운영하며,
AI 학습을 위한 Azure GPU 리소스와 학습 완료된 AI 모델이 서빙된다.
데이터는 MySQL과 Azure DocumentDB에 이중으로 저장되며,
Private Endpoint를 통해 AI 분석 모듈에 안전하게 접근한다.
Azure Monitor로 전체 플랫폼의 상태를 통합 관찰한다.

[데이터 흐름]
병원 데이터 수집
→ PII Masking (온프레미스)
→ Encrypted Data 전송
→ Load Balancer
→ AKS 마이크로서비스 처리
→ AI 모델 분석
→ Result 반환 (병원)

[보안]
PII Masking, 암호화 전송, VPN Gateway, Private Endpoint를 통해
의료 데이터의 기밀성과 무결성을 보장한다.
