이 다이어그램은 실제 개발 및 DevOps 팀이 참조하는 가장 상세한 수준의
구현 다이어그램으로, 이미지 B의 워크로드 구조를 컴포넌트 단위로 구체화하고
이미지 C의 SW/AI/Shared Subscription 내부를 실제 구성 요소로 채운 설계도이다.

[고객사 내부망 - 온프레미스]
납품 기기 또는 고객 내부 시스템(Windows Server) 위에
Medical AI와 동일한 OS 환경의 Docker Container가 구동된다.

ECG Raw Data는 두 가지 경로로 수집된다.
- Real-time Data Container: 실시간 스트리밍 데이터 처리
- File Format Data Container: 배치/파일 단위 데이터 처리

두 경로의 데이터는 PII Masking Container(Python, Java 구현)를 통해
비식별화 처리되며, 처리 중간 데이터는 Volume에 임시 저장된다.

비식별화 완료 데이터는 RabbitMQ Producer를 통해
MQ Format(실시간 스트리밍)과 File Format(배치 전송)으로 구분되어
Load Balancer로 전송된다.

[타 클라우드/On-prem 연동]
이미지 B에서 명시되지 않은 멀티클라우드 데이터 유입 경로가 추가된다.
- AWS DataSync: AWS 환경 데이터 동기화
- Azure Data Factory: GCP, RDBMS 등 이기종 데이터 소스 연동
- Azure File Sync: 온프레미스 파일 시스템 동기화
위 세 경로는 모두 VPN Gateway를 통해 Medical AI Azure로 연결된다.

[Medical AI Azure - SW Subscription]
AKS: 마이크로서비스 실행 환경
Azure VM: RabbitMQ 브로커 운영
MySQL: 서비스 운영 데이터 저장 (사용자/기관/결제 정보)
CosmosDB(MongoDB): ECG 분석 데이터 저장

[Medical AI Azure - AI Subscription]
Private Endpoint: AI 분석 모듈에 대한 외부 직접 접근 차단
AI 분석 모듈: ECG 데이터에 대한 AI 추론 수행
AI Subscription은 복수의 인스턴스로 확장 가능하다.

[Medical AI Azure - Shared Subscription]
Azure Monitor: 전체 플랫폼 통합 모니터링
Key Vault: 비밀키 및 인증서 중앙 관리
AKS Controller: 다중 AKS 클러스터 통합 관리
Storage Account: 공용 스토리지 및 분석 결과 저장

[데이터 흐름 요약]
ECG Raw Data (온프레미스)
→ Real-time / File Format Container 분리 수집
→ PII Masking Container (비식별화)
→ RabbitMQ Producer (MQ Format / File Format)
→ Load Balancer
→ AKS 마이크로서비스
→ CosmosDB(MongoDB) 저장
→ AI 분석 모듈 (Private Endpoint 경유)
→ 분석 결과 저장
→ 사용자 웹 페이지 결과 확인
