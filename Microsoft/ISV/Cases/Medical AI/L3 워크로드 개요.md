이 다이어그램은 MedicalAI Tenant 내에서 운영되는 ECG(심전도) 데이터
AI 분석 서비스의 전체 워크로드 흐름을 나타낸다.
이미지 A보다 한 단계 상세한 수준으로, 실제 서비스 컴포넌트와
데이터 이동 경로를 구체적으로 표현한다.

[데이터 입력 - Customer Tenant]
사용자는 FTP 또는 HTTP를 통해 ECG Raw Data를 업로드한다.
Customer Tenant 내부의 Windows Server에서
PII Encryption 및 Anonymization ��리가 수행되며,
처리된 데이터는 XML, DICOM, IMAGE, PDF 포맷으로 변환된다.
Customer Tenant와 MedicalAI Tenant는 VNet Peering으로 직접 연결되고,
외부 클라우드 및 온프레미스 환경은 VPN Gateway를 통해 연결된다.

[MedicalAI Tenant - 컴퓨팅]
Platform AKS는 다수의 AKS 서비스 배포 상태를 모니터링하고 관리하는
컨트롤러 역할을 수행한다.
Service AKS 위의 마이크로서비스들이 실제 비즈니스 로직을 처리하며,
RabbitMQ Clusters(VM 기반)를 통해 비동기 메시지를 송수신한다.
고객 요구에 따라 전용 Service AKS를 별도 프로비저닝할 수 있으며,
이 경우 전용 데이터베이스도 함께 배포된다.

[MedicalAI Tenant - 데이터 저장]
비식별화된 ECG 데이터는 MongoDB Clusters에 저장된다.
서비스 운영 데이터(사용자/기관/결제 정보)는 MySQL Flexible Server에 저장된다.

[AI 분석]
Azure Container Apps Environment 내의 AI Analysis Module이
MongoDB에 저장된 비식별 ECG 데이터를 분석하며,
Private Endpoint를 통해서만 접근 가능하다.
분석 결과는 ACA Volumes에 저장된다.

[결과 제공]
최종 분석 결과는 사용자가 웹 페이지에서 확인할 수 있으며,
Client System Integration을 통해 고객사 자체 시스템과도 연동된다.

[모니터링]
Azure Monitor와 Grafana를 이중으로 구성하여
서비스 상태 시각화 및 알림을 수행한다.

[네트워크 및 보안]
DNS Zone은 Private/Public 이중 구성으로 운영된다.
모든 AI 분석 모듈 접근은 Private Endpoint를 통해서만 허용된다.
VPN Gateway는 타 클라우드 및 온프레미스 연결 시 사용된다.
