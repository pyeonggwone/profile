이 다이어그램은 MedicalAI 플랫폼 전체를 담는 Azure 구독 구조와
네트워크 거버넌스 설계를 나타낸다.
이미지 A, B의 모든 서비스가 실제로 배치되는 클라우드 인프라 골격이며,
Hub-and-Spoke 아키텍처 기반 Landing Zone 위에 구축된다.
운영 효율성과 접근 제어를 위해 AI 환경과 SW 환경을 별도의
Azure 구독(Subscription)으로 분리하며,
고객별 전용 환경이 필요한 경우 추가 구독을 프로비저닝하여 리소스를 격리한다.

[Hub Subscription]
플랫폼의 네트워크 중심점으로, 모든 트래픽이 이곳을 경유한다.
VPN Gateway를 통해 고객 온프레미스 및 외부 클라우드와 보안 연결하고,
VNet Peering을 통해 각 Spoke 구독과 내부 네트워크를 연결한다.

[Shared Subscription]
플랫폼 전체가 공통으로 사용하는 인프라를 담당한다.
- Key Vault: 비밀키, 인증서, 시크릿 중앙 관리
- Storage Account: 공용 스토리지 제공
- Monitoring: 통합 모니터링 및 관찰성 확보
- Platform AKS Controller: 다수의 AKS 서비스 배포 상태 통합 관리

[AI Subscription + SW Subscription (Spoke, Set 1/2/3)]
고객 및 서비스 단위로 AI 구독과 SW 구독이 쌍을 이루어 구성된다.
AI Subscription: Private Endpoint와 AKS 기반 AI 추론 환경 담당
SW Subscription: 애플리케이션 서비스, MySQL, MongoDB 담당

Subscription Set 1은 고객 전용 환경으로 추가 VM/서버가 포함된다.
Subscription Set 2, 3은 표준 구성으로 운영된다.
신규 고객 전용 환경이 필요한 경우 Subscription Set을 추가하여
완전한 테넌트 격리가 가능하다.

[고객 연결]
Customer Onpremise: VPN Gateway를 통한 암호화 연결
Customer VNET: VNet Peering을 통한 직접 내부 연결

[구독 간 매핑]
이미지 A/B의 AKS → AI Subscription 내 AKS
이미지 A/B의 MySQL, MongoDB → SW Subscription 내 DB
이미지 A/B의 Azure Monitor → Shared Subscription 내 Monitoring
이미지 A/B의 VPN GW → Hub Subscription 내 VPN Gateway
