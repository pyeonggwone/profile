# SQL Server 라이선스 트랙 권장 — CSP Perpetual

> 본 문서는 정확한 견적이 아니라 트랙 결정을 위한 **참고 의견**입니다. 본문 내 수치·제품 정보는 모두 **예시값**이며, 실제 견적은 호스트 코어 수·가상화 여부·인스턴스 수가 확정된 후 다시 산정해야 합니다.

---

## 1. 결론

**CSP Perpetual로 제안하는 것을 권장합니다.**

네이버클라우드 Bare Metal Server에 설치형 SQL Server를 구성하는 본 케이스에서는, **OV L&SA가 아닌 CSP Perpetual** 한 트랙으로 진행하는 것이 가장 현실적이고 비용 효율적입니다.

---

## 2. 권장 근거

### 2-1. License Mobility through SA의 직접 효익이 없음

- Microsoft가 공시하는 **Listed Provider**는 현재 **Microsoft / AWS / Google / Alibaba** 4개사로 한정됩니다.
- **네이버클라우드는 Listed Provider에 포함되어 있지 않으므로**, SA의 핵심 권한인 **License Mobility through SA**는 본 환경에 적용되지 않습니다.
- 즉, OV L&SA로 SA를 추가 구매해도 네이버클라우드 환경에서는 그 권한을 활용할 수 없습니다.

### 2-2. Bare Metal = Dedicated HW → Outsourcing Software Management로 충분

- 네이버클라우드 Bare Metal은 **단일 테넌트 전용(Dedicated) 하드웨어**로 제공됩니다.
- Microsoft Product Terms의 **Outsourcing Software Management** 조항은 Authorized Outsourcer의 **전용 하드웨어** 위에 perpetual 라이선스 설치를 허용합니다.
- 따라서 SA 없이도 CSP Perpetual 라이선스를 합법적으로 운영할 수 있습니다.

### 2-3. 파트너 채널과 정합

- 고객이 네이버클라우드 파트너 채널을 통해 인프라를 도입하므로, **NCE/CSP Perpetual** 흐름이 견적·청구·갱신 과정에서 가장 자연스럽습니다.
- OV L&SA는 별도 Volume Licensing 계약 인프라가 필요해 신규 도입 고객에게는 진입 장벽이 큽니다.

### 2-4. 비용·계약 부담이 작음

- CSP Perpetual은 **1회성 구매**로 종료되며, 연간 SA 갱신 의무가 없습니다.
- 추후 SA가 필요한 시점에만 **CSP Software Subscription(SA add-on)** 으로 분리 추가할 수 있어 유연성이 큽니다.

---

## 3. CSP Perpetual로 충분한 시나리오 vs SA가 필요한 시나리오

| 항목 | CSP Perpetual로 충분 | SA 필요 (OV L&SA 또는 CSP+SA add-on) |
|---|---|---|
| 호스팅 환경 | 네이버클라우드 Bare Metal(전용 HW) 유지 | 향후 Listed Provider(Azure 등) 공유 인프라로 이전 |
| 운영 버전 | 동일 SQL Server 버전 장기 운영 | 미래 상위 버전 사용권 필요 |
| HA 구성 | 단일 인스턴스 또는 동등 라이선스 보유 보조 노드 | AlwaysOn AG/FCI에서 Passive Secondary 무상 운영 |
| 운영 기간 | 2~3년 이상 동일 환경 | 잦은 환경/버전 전환 |

→ 본 케이스의 현재 정보(네이버클라우드 Bare Metal 단일 테넌트, 관리형에서 설치형으로 전환)는 **왼쪽 열에 모두 해당**합니다.

---

## 4. 권장 견적 라인 (예시)

> 아래 항목은 모두 **예시값**입니다. 실제 견적은 호스트 코어 수·가상화 여부·인스턴스 수가 확정된 후 다시 산정합니다.

| 항목 | 예시값 |
|---|---|
| 라이선스 트랙 | CSP Perpetual (NCE) |
| 제품 | SQL Server 2022 Standard Core |
| 라이선스 모델 | Per-Core |
| 최소 단위 | 물리 프로세서당 4 cores (2-core pack 단위 구매) |
| 가정 호스트 코어 수 | 16 cores |
| 가정 인스턴스 수 | 1 |
| 구매 단위 | 2-core pack × 8 = 16 cores |
| 호스팅 | 네이버클라우드 Bare Metal Server (Dedicated single-tenant) |
| Software Assurance | 미포함 (Outsourcing Software Management 조항 적용) |
| Failover 권한 | 없음 (필요 시 SA add-on 별도) |
| 버전 업그레이드 권한 | 없음 (현 버전으로 영구 사용) |

### 옵션 라인 (필요 시 추가)

| 항목 | 예시값 |
|---|---|
| 추가 트랙 | CSP Software Subscription — SQL Server SA add-on |
| 적용 시점 | 상위 버전 사용 또는 AlwaysOn Passive Secondary 무상 운영 필요 시 |
| Term | 36 months / Annual billing |

---

## 5. 주의 사항

- **Bare Metal이 단일 테넌트 전용 HW**라는 전제가 깨지면(공유 가상화 풀로 변경 등) 본 권장은 무효가 됩니다. 견적서/계약서에 *"Customer-dedicated single-tenant hardware"* 조건 명시 권장.
- **AlwaysOn AG / FCI 구성**이 들어가면 Passive Secondary 노드 운영을 위해 SA가 필요해질 수 있습니다. HA 구성이 확정되는 시점에 SA add-on 라인을 별도 산정하면 됩니다.
- **License Mobility through SA**는 네이버클라우드에서는 효익이 없습니다. 본 환경에서 SA의 가치는 **버전 업그레이드 권한 + Failover Rights**로만 평가하면 됩니다.
