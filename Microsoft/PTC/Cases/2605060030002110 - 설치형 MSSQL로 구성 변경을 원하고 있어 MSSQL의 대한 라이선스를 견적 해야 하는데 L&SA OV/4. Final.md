# 네이버클라우드 Bare Metal Server SQL Server 라이선스 견적 회신

## 1. 결론

**CSP Perpetual로 제안하는 것을 권장합니다.**

네이버클라우드 Bare Metal Server에 설치형 SQL Server를 구성하는 본 케이스에서는, **OV L&SA가 아닌 CSP Perpetual** 한 트랙으로 진행하는 것이 가장 현실적이고 비용 효율적입니다.
---

## 2. 질문별 답변

### Q1. CSP Perpetual로 제안해도 되는지, L&SA OV가 필수인지

**답변: 가능하나 제약 있음.** 네이버클라우드 Bare Metal Server가 **단일 테넌트 전용(Dedicated) 하드웨어**로 제공되는 한, CSP Perpetual 제안으로도 라이선스 적용은 가능합니다. 다만 이 경우 Microsoft Product Terms의 **Outsourcing Software Management** 조항이 적용되며, 운영 중 발생하는 일부 권한(상위 버전 사용권, License Mobility, Failover 권한)이 제한됩니다.

**권장 설계**

- 워크로드를 **단일 테넌트 전용 Bare Metal**로 고정하고, 호스트 단위 코어를 모두 라이선스합니다. SQL Server는 **per-core** 모델로 산정하며 물리 서버 또는 VM 당 **최소 4코어**가 필요합니다.
- 향후 버전 업그레이드(예: SQL Server vNext 출시 시), 다른 호스팅 환경으로의 이동, AlwaysOn 보조 노드의 Failover 권한이 필요하면 **L&SA OV** 또는 **CSP Perpetual + SA add-on** 으로 SA를 함께 확보합니다.
- 단기간(2~3년) 동일 버전 운영이 확실한 경우에만 **CSP Perpetual 단독**을 선택지에 포함합니다.

**구현 방법**

- 기존 라이선스 / SA 상태 점검: Volume Licensing Service Center > Licenses > License Summary에서 Agreement 단위(예: Open Value `V8923741`)로 SQL Server SKU의 SA Coverage Start/End를 펼쳐 본다. 활성 SA가 없거나 만료(예: `2027-03-31`)가 가까우면 두 트랙(With SA / Without SA) 견적을 모두 준비한다.
- CSP Perpetual 견적 라인: Partner Center > Pricing and offers > Perpetual Software 카탈로그에서 `SQL Server 2022 Standard Core`(SKU `DG7GMGF0M7XW:0002`)를 선택한 뒤, 대상 고객(예: `한라테크 주식회사`, Tenant ID `7a3c2f1e-8d4b-4e9a-9f6c-2b1d5e8a7c3f`)에 New Subscription > Perpetual Software 흐름으로 **2-core pack × 4(=8 cores)** 라인을 pilot 견적(`Hallatech-SQLStd-Core-2026Q2`)으로 Save as Draft 한다.
- 배포 메타데이터 입력: 동일 Configuration 화면에 `Deployment Target: Naver Cloud Bare Metal Server`, `Server Hostname: hallatech-sqlbm-prd-01`, `Region: KR-Pangyo IDC`, `Hardware Type: Dedicated single-tenant` 를 명시해 Outsourcing 조건이 견적 라인에 기록되도록 한다.

**주의**

- 네이버클라우드는 현재 Microsoft가 발표한 **Listed Provider**(Microsoft / AWS / Google / Alibaba) **목록에 포함되어 있지 않습니다**. Bare Metal이 **공유(멀티테넌트) 자원**으로 제공되거나, 향후 가상화 공유 풀로 변경되면 SA 없는 perpetual 라이선스로는 운영할 수 없습니다.
- SA가 없으면 **버전 업그레이드 권한 없음**, **License Mobility through SA 적용 불가**, **Passive Secondary Failover 권한 없음** 입니다.
- pilot 견적은 Draft 상태로만 생성하고, 실제 주문 확정 전에 고객값(코어 수, 에디션, HA 토폴로지)으로 라인을 다시 검산한다.

**reference**

- Microsoft Product Terms: *"Customer may install and use licensed software on Outsourcer's servers ... provided that the servers are dedicated to Customer's use."* — https://www.microsoft.com/licensing/docs/view/Product-Terms
- SQL Server 라이선싱 가이드: per-core 라이선스의 최소 단위는 물리 코어 또는 가상 코어 4개 — https://www.microsoft.com/sql-server/sql-server-2022

### Q2. SA(Software Assurance) 보유가 필수인지

**답변: 가능 / 시나리오별 분기.** 네이버클라우드 Bare Metal이 고객 전용 단일 테넌트 하드웨어이고 동일 버전을 유지한다면 **SA는 필수가 아닙니다**. 그러나 아래 시나리오 중 하나라도 해당되면 **SA가 사실상 필수**입니다.

**권장 설계**

- SA 필수 시나리오
  1. 미래 버전 업그레이드(예: SQL Server vNext) 사용을 계획
  2. 동일 라이선스를 **다른 호스팅 환경**(특히 Listed Provider의 공유 인프라)으로 이동할 계획
  3. AlwaysOn AG/FCI에서 **Passive Secondary Failover 노드를 무상 라이선스로 운영**
  4. 분기·연 단위 **사용량 변동**이 큰 환경에서 라이선스 유연성 확보가 필요
- 위 중 하나라도 해당되면 **L&SA OV** 또는 **CSP Perpetual + SA add-on** 형태로 제안합니다. 신규 도입 + 단순 단일 인스턴스 + 동일 버전 장기 운영이면 **CSP Perpetual 단독** 도 적법한 선택지입니다.

**구현 방법**

- SA add-on 라인 추가: Partner Center > Pricing and offers > Software Subscriptions에서 `SQL Server 2022 Standard Core SA` 를 검색해 기존 Draft 견적에 **2-core pack × 4(=8 cores)** SA 라인을 추가하고, 합산본을 `Hallatech-SQLStd-Core-2026Q2-with-SA` 이름으로 Save as Draft 한다. Term은 36 months / Annual billing을 기준으로 한다.
- L&SA OV 트랙: 기존에 Open Value 계약(예: `V8923741`)이 있다면 동일 Agreement에 SQL Server Standard Core 라이선스 + SA를 추가하고, License Summary에서 신규 SA Coverage가 정상 발급되는지 본다.
- License Mobility 사전 등록(필요 시): VLSC > Software Assurance Benefits에서 License Mobility Verification Form을 다운로드해 `Authorized Mobility Partner: 네이버클라우드 주식회사`, `Hosting Type: Dedicated Bare Metal Server`, `Effective Date: 2026-06-01` 로 작성 후 Software Assurance > License Mobility 화면에 업로드(`Submission Reference: LM-2026-HALLATECH-01`). 이 작업은 향후 Listed Provider 공유 호스팅으로 옮겨갈 가능성을 대비한 pilot 등록이며, 현 네이버클라우드 단일 테넌트 운영에는 직접 적용되지 않는다.

**주의**

- **License Mobility through SA**는 **Listed Provider의 공유 인프라**로 이동할 때만 의미가 있습니다. 네이버클라우드는 Listed Provider가 아니므로 이 권한은 현재 단계에서 직접적인 효익이 없습니다.
- **Outsourcing Software Management** 조항은 **Authorized Outsourcer의 전용 하드웨어**에 한정되므로, 호스팅 형태가 변경되는 즉시 적용 조건이 깨집니다.
- SA 추가 라인은 견적 단계에서는 옵션 비교용으로만 보유하고, 고객이 시나리오 중 어느 것에 해당하는지 답이 정해진 뒤에 결합/단독을 결정한다.

**reference**

- Microsoft License Mobility through SA — https://www.microsoft.com/licensing/licensing-programs/software-assurance-license-mobility
- Software Assurance Benefits(신규 버전 사용권, Failover Rights, License Mobility) — https://www.microsoft.com/licensing/licensing-programs/software-assurance-by-product

## 3. 권장 결론(요약)

| 시나리오 | 권장 라이선스 | SA 필요성 |
| --- | --- | --- |
| 단일 테넌트 Bare Metal, 동일 버전 장기 운영, 이동 계획 없음 | CSP Perpetual | 선택 사항 |
| 버전 업그레이드 또는 호스팅 이동 가능성 있음 | L&SA OV 또는 CSP Perpetual + SA add-on | 필수 |
| AlwaysOn AG/FCI Secondary 무상 운영 필요 | L&SA OV 또는 CSP Perpetual + SA add-on | 필수 |
| 향후 멀티테넌트(공유) 호스팅으로 이전 가능성 있음 | L&SA OV(우선) | 필수 |

**가장 안전한 기본 제안:** CSP Perpetual 가격을 기본선으로 두고, SA add-on 라인을 함께 옵션 견적에 포함해 고객이 비교 결정할 수 있도록 두 가지 라인 모두 제출하는 방식.

## 4. Best Practice

- SQL Server 코어 산정은 **물리 서버 전체 코어 수 또는 VM 가상 코어 수 중 큰 값**을 기준으로 한다. 최소 라이선스 단위는 4코어, **2-core pack** 단위로 구매한다.
- AlwaysOn AG/FCI 구성 시 **Passive Secondary Failover Rights**(SA 보유 시 1회 부여)를 활용해 라이선스 비용을 절감한다.
- Bare Metal 호스트 사양·코어 수는 변경되더라도 라이선스 산정에 직접 영향을 미치므로, **고정된 코어 수의 SKU**를 선정해 호스트와 매칭시킨다.
- 향후 Listed Provider(예: Azure)로 마이그레이션 가능성이 조금이라도 있다면 SA를 처음부터 포함한다. 사후 추가 시 비용/계약 부담이 더 크다.

## 6. Pilot 흐름 요약(7단계 이내)

1. Volume Licensing Service Center에서 고객 Open Value Agreement(예: `V8923741`)와 SQL Server SA Coverage 상태를 점검한다.
2. Partner Center Perpetual Software 카탈로그에서 `SQL Server 2022 Standard Core`(SKU `DG7GMGF0M7XW:0002`)를 선택해 CSP Perpetual 기준선 견적(`Hallatech-SQLStd-Core-2026Q2`)을 Draft로 만든다.
3. 동일 Draft에 배포 메타데이터(`Deployment Target: Naver Cloud Bare Metal Server`, `Hardware Type: Dedicated single-tenant`)를 기록한다.
4. Software Subscriptions에서 SA add-on 라인(`SQL Server 2022 Standard Core SA`)을 추가한 결합본(`Hallatech-SQLStd-Core-2026Q2-with-SA`)을 별도 Draft로 보존해 두 트랙 비교가 가능하도록 한다.
5. 향후 Listed Provider 이전 가능성에 대비해 License Mobility Verification Form을 report-only 성격의 pilot 등록(`Submission Reference: LM-2026-HALLATECH-01`)으로 준비한다.
6. Microsoft Product Terms의 Outsourcing Software Management 조항과 Listed Providers 페이지를 근거로 두 트랙의 적용 조건을 견적서 부속 메모로 정리한다.
7. 고객으로부터 입력값을 회수해 코어 수, 에디션, HA 토폴로지, 운영 기간을 확정한 뒤 두 Draft 중 하나를 본 견적으로 승격시킨다.

## 7. 설계 확정에 필요한 입력값

| 항목 | 요청 사유 |
| --- | --- |
| Bare Metal 하드웨어가 **단일 테넌트 전용** 인지 | Outsourcing Software Management 적용 가능 여부 판단 |
| 호스트 **물리 코어 수** 및 가상화 사용 여부 | per-core 라이선스 수량 산정 |
| SQL Server **에디션**(Standard / Enterprise) 및 **버전** | SKU/SA add-on 가격 산정 |
| **HA 토폴로지**(단일 / AlwaysOn AG / FCI) 및 Secondary 노드 수 | Passive Failover Rights 필요 여부 |
| 고객의 **기존 VL 계약 / 활성 SA 보유 여부** | L&SA OV 신규/갱신 또는 CSP 신규 결정 |
| **운영 기간** 및 향후 버전 업그레이드/이전 계획 | SA 필수 시나리오 적용 여부 |
