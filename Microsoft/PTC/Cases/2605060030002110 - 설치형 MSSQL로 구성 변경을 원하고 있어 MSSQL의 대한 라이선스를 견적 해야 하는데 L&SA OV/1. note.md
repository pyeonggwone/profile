## 3. Scoping
- Licensing model comparison
  - L&SA via Open Value: perpetual license + SA, includes License Mobility through SA and version upgrade rights
  - CSP Perpetual (NCE): perpetual license without SA by default; SA available only as separate subscription add-on
- Deployment topology: SQL Server installed by the customer on Naver Cloud Bare Metal Server (third-party infrastructure, dedicated HW assumed)
- Outsourcing rules
  - Listed Providers (Microsoft, AWS, Google, Alibaba) require License Mobility through SA for shared infrastructure
  - Non-Listed outsourcers fall under Outsourcing Software Management terms; perpetual licenses may run on dedicated outsourcer hardware
- Core licensing
  - Per-core model, minimum 4 cores per physical server / VM
  - Edition decision (Standard vs Enterprise) driven by HA, scale, and feature requirements
- Mobility and continuity
  - License Mobility through SA needed to move existing licenses to shared/multi-tenant hosting
  - Without SA, perpetual licenses are bound by Outsourcing terms and dedicated-HW constraints

## 4. Requirement Evaluation
1. CSP Perpetual로 구매한 SQL Server 라이선스를 네이버클라우드 Bare Metal Server에 설치/운영할 수 있는지
2. L&SA OV(Open Value + Software Assurance) 제안이 필수인지, SA 없이도 합법적 운영이 가능한지
3. License Mobility through SA 적용 가능 여부 및 네이버클라우드의 Listed Provider 등재 여부
4. Outsourcing Software Management(Authorized Outsourcer) 규정에 따른 Bare Metal(전용 하드웨어) 적용 조건
5. SQL Server 코어 라이선스 산정 기준(per-core 최소 4코어, 에디션 선택)
6. 권장 라이선스 옵션 및 향후 SA 추가/갱신, NCE 전환 시 고려사항

## 6. Research Notes
- Summary:
  - The customer must decide between L&SA OV (perpetual + SA) and CSP Perpetual (no SA) for SQL Server installed on Naver Cloud Bare Metal Server, and whether SA is mandatory.
- Facts:
  - Microsoft Product Terms classify Listed Providers (Microsoft, AWS, Google, Alibaba); shared infrastructure on Listed Providers requires License Mobility through SA.
  - Non-Listed outsourcers (e.g., Naver Cloud, currently not on the Listed Provider list) fall under Outsourcing Software Management; perpetual licenses can be deployed on dedicated outsourcer hardware without SA, subject to those terms.
  - SQL Server uses per-core licensing with a 4-core minimum per physical server / VM; edition (Standard / Enterprise) drives feature and HA scope.
  - License Mobility through SA, Failover Rights, and version upgrade rights all require active Software Assurance.
  - CSP Perpetual (NCE) does not include SA by default; SA must be added as a separate subscription if mobility/upgrade rights are needed.
- Follow-up Questions:
  - Is the Naver Cloud Bare Metal Server fully dedicated single-tenant hardware (no virtualization sharing)?
  - Does the customer hold any existing Volume Licensing agreement or active SA?
  - Target SQL Server edition, total core count, and HA topology (AlwaysOn AG, FCI, standalone)?
