## Input

- `reference/case_statement.md`
- 보조 참고 자료: `reference/` 폴더 내 문서 전체
- 기준 범위: `case_statement.md`의 `[Customer Statement]`  프로젝트 작업 정리 내용

## Task

1. `case_statement.md`의 `[Customer Statement]`를 읽고 고객 문의의 기본 맥락을 파악한다.
2. `reference/` 폴더의 문서는 보조로 참고하여 이 프로젝트에서 이미 정리된 작업 맥락, 해결 방향, 검토 항목을 파악한다.
3. 고객 문의 원문과 프로젝트 작업 정리 내용을 종합하여 조사해야 할 핵심 항목을 식별한다.
4. 각 항목을 이후 `jsonl/note_templete.jsonl` 기준으로 `pre_scoping`, `scoping`, `research_notes`를 작성할 수 있는 독립적인 검토 주제로 재구성한다.
5. 각 항목은 최종 고객 답변에서 사용할 수 있는 대분류 제목 역할도 해야 한다.
6. 원문 문장이나 reference 문장을 그대로 재사용하지 않고, 의미를 요약 및 재해석한다.
7. `case_statement.md`와 `reference/` 폴더 어느 쪽에도 근거가 없는 조사 항목이나 요구사항은 임의로 추가하지 않는다.

## Output

- 파일명: `1. problem list.md`
- 형식: 번호가 매겨진 리스트
- 언어: 한국어
- 내용: 조사를 위한 항목이자, 항목별 답변 작성을 위한 대분류 제목만 작성
- 작성 기준:
	- 각 항목은 독립적으로 `pre_scoping`, `scoping`, `research_notes`를 작성할 수 있어야 한다.
	- 각 항목은 너무 세부적인 실행 단계가 아니라 답변을 구조화할 수 있는 상위 검토 주제여야 한다.
	- 필요 시 기술 검토, 절차 검토, 비용/일정 검토, 내부 에스컬레이션, 지원 범위 경계를 구분한다.
- 제외: 설명, 배경, 분석 과정, 요약 문단
