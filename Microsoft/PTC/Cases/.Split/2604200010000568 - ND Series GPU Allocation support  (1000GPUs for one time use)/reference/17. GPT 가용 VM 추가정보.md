# 추가 정보 기준 가용 VM 리스트

## 검증 결과 요약

현재 정리는 `14. GPU 추가정보.md`, `10. GPU 가용_리전.json`, `10. GPU 가용_VM.json`, `10. VM별상세스펙.json` 기준으로 대체로 맞습니다.

- GPU 수량 산정은 대표 SKU의 최대 GPU 수량 기준으로 계산되어 있습니다.
- 256 GPUs 요청안은 고스펙 128 GPUs + 저스펙 128 GPUs로 나누는 해석이 고객 요청과 일치합니다.
- fallback 128 GPUs 요청안은 고스펙 64 GPUs + 저스펙 64 GPUs로 나누는 해석이 적절합니다.
- 저스펙 GPU 16장 상시 사용 기준도 각 대표 SKU의 GPU/VM 수량으로 나눈 VM 수량이 맞습니다.
- 리전 조합은 공개 availability 데이터 기준의 동일 region 내 series 공존 여부를 본 것이며, 실제 capacity/stock/approval 가능성을 보장하지는 않습니다.

## 보충 검토 포인트

- 고객이 말한 `B/H 계열 GPU` 중 H 계열은 `NDsr H100 v5-Series`, `NDsr H200 v5-Series`로 검토할 수 있습니다. 다만 현재 로컬 JSON에는 B 계열 GPU SKU가 없으므로, B 계열이 필수라면 Microsoft capacity team 또는 Azure Support에 별도 확인이 필요합니다.
- 고스펙 GPU 간 RDMA/InfiniBand가 필수인 요구사항에는 `NDsr H100 v5-Series`, `NDsr H200 v5-Series` 중심 검토가 타당합니다.
- `NCadsH100_v5-series`, `NCCadsH100_v5-series`도 H100 계열이지만 현재 문서의 고스펙 RDMA 기준에서는 제외했습니다. PCIe H100 또는 confidential H100 성격의 대안으로 별도 검토는 가능하지만, 고객 요구가 대규모 RDMA/InfiniBand 분산학습이면 `NDsr` 계열을 우선 확인하는 것이 안전합니다.
- 케이스 2는 저스펙 RDMA GPU가 없는 리전에서 일반 GPU를 조합하는 대안입니다. 저스펙 GPU까지 RDMA가 필요하다고 해석되면 케이스 2는 제외하고 케이스 1 또는 케이스 3 중심으로 봐야 합니다.
- 동일 Kubernetes cluster 편입은 가능하더라도, RDMA가 필요한 training job은 RDMA GPU node pool에 node selector/taint/toleration 등으로 묶어 배치하는 전제가 필요합니다.
- 실제 요청 시에는 VM series만 제시하기보다 preferred period, fallback period, 총 GPU 수, 상시 사용 GPU 수, burst 사용일, 동일 region 조건, RDMA 필수 범위를 함께 전달해야 capacity 검토가 수월합니다.

### 한국 고객 관점의 현실 후보

- Korea Central: `NDsr H100 v5-Series`와 `NCads A100 v4-Series`, `NCas T4 v3-Series`, `NVadsA10_v5-series` 조합 가능. 저스펙 RDMA 후보는 현재 데이터 기준 없음.
- Japan East: `NDsr H100 v5-Series`와 일반 GPU 후보 조합 가능. 한국 인접 리전 대안으로 검토 가능.
- Australia East: `NDsr H100 v5-Series`, `NDsr H200 v5-Series`와 일반 GPU 후보 조합 가능. H100/H200 양쪽을 모두 볼 수 있는 일반 GPU 조합 후보입니다.
- East US 2, West Europe, UK South: 저스펙 RDMA까지 포함한 기술적 후보가 풍부하지만, 고객의 데이터 위치/latency/compliance 조건을 별도 확인해야 합니다.

## 산정 기준

`14. GPU 추가정보.md`의 고객 요구사항과 `10. VM별상세스펙.json`의 GPU 수량을 기준으로 아래처럼 산정했습니다.

- 1순위 요청안: 총 256 GPUs = 고스펙 GPU 128장 + 저스펙 GPU 128장
- 2순위 fallback: 총 128 GPUs = 고스펙 GPU 64장 + 저스펙 GPU 64장
- 상시 사용 저스펙 GPU: 16 GPUs
- 고스펙 GPU는 RDMA/InfiniBand 지원이 필수이므로 `NDsr H100 v5-Series`, `NDsr H200 v5-Series`만 기준으로 사용
- VM 개수는 각 VM series에서 사용 가능한 최대 GPU 수량의 SKU 기준으로 계산
- 동일 Azure region 안에서 고스펙 RDMA VM과 저스펙 GPU VM을 함께 확보할 수 있는 조합만 가용 후보로 정리
