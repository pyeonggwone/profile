## 현 프로젝트 기준 답변 가능 항목

1. 더 낮은 코어 수로 진행 가능한지 여부
	- 답변 가능: `Yes`
	- 근거: 기존 요청은 256 GPU 기준이나, 프로젝트 내 추가 정보에서 256 GPU 확보가 어려운 경우 128 GPU까지 축소 가능하다고 정리되어 있다.

2. 최소 필요 코어 수
	- 답변 가능: 부분 가능
	- `NDasr A100 v4` 기준: `Standard_ND96asr_v4` 16대 = 1,536 vCPU = 128 A100 GPUs
	- `NCads H100 v5` 기준: 확인 후 전달 필요
	- 사유: 프로젝트 내 VM 상세 정보 기준 `NCads H100 v5`는 `Standard_NC40ads_H100_v5` 40 vCPU 또는 `Standard_NC80adis_H100_v5` 80 vCPU 구성이며, 1,536 vCPU와 정확히 매칭되는 VM 수량이 확정되지 않는다.

3. 생성하려는 정확한 VM size
	- 답변 가능: 부분 가능
	- A100 후보: `Standard_ND96asr_v4`
	- H100 후보: 확인 후 전달 필요
	- 사유: 프로젝트 내 H100 검토는 주로 `Standard_ND96isr_H100_v5` 기준으로 정리되어 있으며, 현재 질문의 `NCads H100 v5`와는 series가 다르다.

4. 사용 계획 시작 날짜와 완료 날짜
	- 답변 가능: `2026-08-01 ~ 2026-08-30`
	- 대체 가능 일정: 8월 capacity 확보가 어려운 경우 9월 가능

5. ramp-up 계획
	- 답변 가능
	- 1~2주차: CPU 인스턴스와 소수의 저사양 GPU 인스턴스로 Kubernetes 환경 구성
	- 3~4주차: 격일로 대규모 GPU 테스트 수행
	- 테스트일: 256 GPU 또는 fallback 128 GPU 규모로 분산학습/분산추론 실행
	- 비테스트일: 버그 수정, 환경 조정, 다음 테스트 준비

6. 원 요청 유지가 필요한 business purpose 및 dependency
	- 답변 가능
	- 목적: open-source LLM 및 Acryl 의료특화 파운데이션 모델 `A-LLM.H`의 분산학습/분산추론 검증
	- 목적: `JONATHAN GPUBASE` 적용 전후 성능 향상 검증
	- 의존성: 동일 Azure region 내 CPU/GPU VM을 배치하고 Acryl custom Kubernetes cluster에 편입해야 함
	- 의존성: 고성능 GPU 간 RDMA/InfiniBand가 필요함
	- 비즈니스 영향: GPU capacity 확보가 확정 opportunity 진행의 blocker임

## 확인 후 전달 필요 항목

1. `NCads H100 v5` 1,536 vCPU에 해당하는 정확한 VM size와 VM 수량을 확인 후 전달하라.
2. 고객이 실제로 `NCads H100 v5`를 원하는지, 아니면 프로젝트 내 검토된 `NDsr H100 v5` / `Standard_ND96isr_H100_v5`를 원하는지 확인 후 전달하라.
3. 최종 target region은 capacity team 또는 Azure Support 확인 후 전달하라.

## 지금 확인해서 표시 가능한 항목

1. `NDasr A100 v4` 1,536 vCPU
	- 표시 가능
	- VM size: `Standard_ND96asr_v4`
	- vCPU/VM: 96
	- GPU/VM: 8 A100 GPUs
	- 필요 VM 수량: 16대
	- 총 vCPU: 1,536 vCPU
	- 총 GPU: 128 A100 GPUs
	- RDMA/InfiniBand: 지원

2. `NDasr A100 v4` 후보 리전
	- 표시 가능

	- `East US`
	- `North Central US`
	- `South Central US`
	- `West US 2`
	- `USGov Arizona`
	- `USGov Texas`
	- `China North 2`
	- 실제 capacity 가능 여부는 확인 후 전달 필요

3. `NCads H100 v5` 1,536 vCPU
	- 정확한 표시 불가
	- 프로젝트 내 확인된 VM size는 `Standard_NC40ads_H100_v5`와 `Standard_NC80adis_H100_v5`
	- `Standard_NC40ads_H100_v5`: 40 vCPU, 1 H100 GPU
	- `Standard_NC80adis_H100_v5`: 80 vCPU, 2 H100 GPUs
	- 1,536 vCPU는 40 또는 80 vCPU 단위 VM 조합으로 정확히 산출되지 않음
	- 정확한 VM 수량과 quota family는 확인 후 전달 필요

4. `NCads H100 v5` 대신 프로젝트 내 H100 대체 후보

	- vCPU/VM: 96
	- GPU/VM: 8 H100 GPUs
	- 1,536 vCPU 기준 필요 VM 수량: 16대
	- 총 GPU: 128 H100 GPUs
	- RDMA/InfiniBand: 지원
	- 단, 이는 `NCads H100 v5`가 아니라 `NDsr H100 v5` 기준이므로 고객 의도 확인 후 전달 필요

5. 사용 기간
