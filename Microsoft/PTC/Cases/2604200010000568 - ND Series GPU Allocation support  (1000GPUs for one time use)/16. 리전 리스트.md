# 추가 정보 기준 가용 리전 검토

디테일 공유 감사드립니다.

이번 추가 요청에서는 아래 두 조건을 동시에 만족하는 리전을 찾는 것이 핵심입니다.

- 동일 Azure region 내에서 CPU/GPU VM을 함께 구성할 수 있어야 함
- 모든 VM을 아크릴 커스텀 Kubernetes cluster에 편입할 수 있어야 함

고스펙 GPU는 RDMA/InfiniBand가 필수 조건이므로 `NDsr H100 v5-Series`, `NDsr H200 v5-Series`를 우선 기준으로 보겠습니다. 저스펙 GPU는 RDMA/InfiniBand 충족 여부에 따라 별도 케이스로 나누어 검토했습니다.

참고 기준 파일은 아래 두 개입니다.

- `10. GPU 가용_리전.json`: 리전별 사용 가능한 GPU series
- `10. GPU 가용_VM.json`: GPU series별 사용 가능한 리전

## 전제

### 고스펙 GPU 기준

고스펙 GPU는 분산학습/분산추론에서 GPU 간 통신 성능이 중요하므로 RDMA/InfiniBand 지원 series를 기준으로 봅니다.

- `NDsr H100 v5-Series`
- `NDsr H200 v5-Series`

현재 JSON에는 별도 B-series GPU 항목이 없으므로, B 계열 GPU가 반드시 필요한 경우에는 Microsoft capacity team 또는 Azure Support를 통해 별도 SKU availability 확인이 필요합니다.

### 저스펙 GPU 기준

저스펙 GPU는 두 가지로 나누어 봅니다.

RDMA/InfiniBand 충족 후보:

- `NDasr A100 v4-Series`
- `NDamsr A100 v4-Series`
- `NDv2-Series`

RDMA/InfiniBand 미충족 또는 일반 GPU node 후보:

- `NCads A100 v4-Series`
- `NCas T4 v3-Series`
- `NVadsA10_v5-series`
- `NVads_V710_v5-series`
- `NGads_V620_v1-series`

## 케이스 1. 고스펙 RDMA + 저스펙도 RDMA 충족

가장 좋은 케이스입니다. 고스펙 GPU와 저스펙 GPU 모두 RDMA/InfiniBand 계열 후보가 같은 region 안에 존재합니다.

이 경우 하나의 Kubernetes cluster 안에서 고스펙 GPU node group과 저스펙 GPU node group을 구성하되, 분산학습 job은 RDMA 가능 node group에 배치하는 방향으로 설계할 수 있습니다.

### 후보 리전

#### West Europe

- 고스펙 RDMA: `NDsr H100 v5-Series`
- 저스펙 RDMA: `NDamsr A100 v4-Series`, `NDasr A100 v4-Series`, `NDv2-Series`
- 저스펙 일반 GPU: `NCads A100 v4-Series`, `NCas T4 v3-Series`, `NGads_V620_v1-series`, `NVads_V710_v5-series`, `NVadsA10_v5-series`

#### UK South

- 고스펙 RDMA: `NDsr H100 v5-Series`, `NDsr H200 v5-Series`
- 저스펙 RDMA: `NDasr A100 v4-Series`
- 저스펙 일반 GPU: `NCads A100 v4-Series`, `NCas T4 v3-Series`, `NVadsA10_v5-series`

#### East US

- 고스펙 RDMA: `NDsr H100 v5-Series`
- 저스펙 RDMA: `NDasr A100 v4-Series`, `NDv2-Series`
- 저스펙 일반 GPU: `NCads A100 v4-Series`, `NCas T4 v3-Series`, `NVads_V710_v5-series`, `NVadsA10_v5-series`

#### East US 2

- 고스펙 RDMA: `NDsr H100 v5-Series`, `NDsr H200 v5-Series`
- 저스펙 RDMA: `NDamsr A100 v4-Series`
- 저스펙 일반 GPU: `NCads A100 v4-Series`, `NCas T4 v3-Series`, `NGads_V620_v1-series`, `NVadsA10_v5-series`

#### North Central US

- 고스펙 RDMA: `NDsr H200 v5-Series`
- 저스펙 RDMA: `NDasr A100 v4-Series`
- 저스펙 일반 GPU: `NCads A100 v4-Series`, `NCas T4 v3-Series`, `NVads_V710_v5-series`, `NVadsA10_v5-series`

### 판단

위 5개 리전이 가장 우선 검토 대상입니다.

특히 `East US`, `East US 2`, `West Europe`, `UK South`는 고스펙 RDMA GPU와 저스펙 GPU 후보가 함께 풍부하므로, capacity 확인 요청 시 1순위 후보로 올리기 좋습니다.

## 케이스 2. 고스펙 RDMA + 저스펙은 일반 GPU로 구성

이 케이스는 고스펙 GPU에는 RDMA/InfiniBand를 적용하고, 저스펙 GPU는 일반 GPU node로 구성하는 방식입니다.

Kubernetes cluster는 동일 region 안에서 구성할 수 있지만, 저스펙 GPU node가 RDMA fabric에 참여한다고 보기는 어렵습니다. 따라서 RDMA가 필요한 분산학습은 고스펙 GPU node group에서 실행하고, 저스펙 GPU는 preprocessing, inference, bug fix, 테스트 준비, 보조 workload에 배치하는 방식이 적합합니다.

### 후보 리전

#### South Africa North

- 고스펙 RDMA: `NDsr H100 v5-Series`, `NDsr H200 v5-Series`
- 저스펙 일반 GPU: `NVadsA10_v5-series`

#### Australia East

- 고스펙 RDMA: `NDsr H100 v5-Series`, `NDsr H200 v5-Series`
- 저스펙 일반 GPU: `NCads A100 v4-Series`, `NCas T4 v3-Series`, `NVadsA10_v5-series`

#### USGov Arizona

- 고스펙 RDMA: `NDsr H100 v5-Series`
- 저스펙 일반 GPU: `NVadsA10_v5-series`

#### France Central

- 고스펙 RDMA: `NDsr H200 v5-Series`
- 저스펙 일반 GPU: `NCads A100 v4-Series`, `NVadsA10_v5-series`

#### Italy North

- 고스펙 RDMA: `NDsr H200 v5-Series`
- 저스펙 일반 GPU: `NCads A100 v4-Series`, `NCas T4 v3-Series`, `NVadsA10_v5-series`

#### Japan East

- 고스펙 RDMA: `NDsr H100 v5-Series`
- 저스펙 일반 GPU: `NCads A100 v4-Series`, `NCas T4 v3-Series`, `NVadsA10_v5-series`

#### Japan West

- 고스펙 RDMA: `NDsr H200 v5-Series`
- 저스펙 일반 GPU: `NCas T4 v3-Series`

#### Korea Central

#### West US

### 판단

이 케이스는 저스펙 GPU까지 RDMA가 필요하지 않다면 충분히 검토 가능합니다.

다만 고객 요구사항이 “모든 GPU node 간 RDMA”로 해석될 경우에는 적합하지 않습니다. 요구사항을 “고스펙 GPU 간 RDMA 필수, 저스펙 GPU는 같은 Kubernetes cluster 참여 가능”으로 정리하면 이 케이스도 현실적인 대안이 됩니다.

## 케이스 3. 고스펙 RDMA + 저스펙 RDMA/일반 GPU 모두 선택 가능

이 케이스는 케이스 1의 리전 중에서도 저스펙 GPU 선택지가 더 넓은 리전입니다. capacity 확보 가능성을 높이기 위해 Support/UAT 요청 시 우선 후보로 제시할 수 있습니다.

### 우선 후보

#### West Europe

- 고스펙 RDMA와 저스펙 RDMA 후보가 모두 있음
- 저스펙 일반 GPU 후보도 가장 다양함
- 가장 유연한 후보 중 하나

#### East US

- 고스펙 RDMA와 저스펙 RDMA 후보가 모두 있음
- 기존 케이스에서도 주요 후보로 검토된 리전
- 일반 GPU 후보도 함께 존재

#### East US 2

- H100/H200 모두 보이고, 저스펙 RDMA 후보도 있음
- 일반 GPU 후보도 다양함

#### UK South

- H100/H200 모두 보이고, A100 RDMA 후보도 있음
- 일반 GPU 후보도 함께 존재

#### North Central US

- H200 + A100 RDMA 조합 가능
- 일반 GPU 후보도 함께 존재

### 고객에게 설명할 핵심 메시지

고스펙 GPU에는 RDMA/InfiniBand가 필수이므로 `NDsr H100 v5-Series` 또는 `NDsr H200 v5-Series`가 있는 리전을 먼저 확인해야 합니다. 저스펙 GPU까지 RDMA가 필요하면 후보 리전은 `West Europe`, `UK South`, `East US`, `East US 2`, `North Central US`로 좁혀집니다.

반대로 저스펙 GPU가 같은 Kubernetes cluster에만 참여하면 되고 RDMA가 필수가 아니라면, `NCads A100`, `NCas T4`, `NVadsA10` 계열을 활용할 수 있어 선택 가능한 리전이 크게 늘어납니다. 이 경우 RDMA가 필요한 분산학습 job은 고스펙 GPU node group에만 배치하고, 저스펙 GPU는 보조 workload나 테스트 준비 용도로 분리하는 구성이 적합합니다.
