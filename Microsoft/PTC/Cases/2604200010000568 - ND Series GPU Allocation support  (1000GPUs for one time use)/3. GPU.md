# GPU 확보 가이드 — ND Series (InfiniBand, Azure)

---

## 1. GPU 확보 전제 조건

ND 시리즈 GPU를 Azure에서 확보하려면 아래 조건이 모두 충족되어야 합니다.

| 조건 | 내용 |
|---|---|
| **유효한 Azure 구독** | EA, CSP, PAYG 모두 가능. 무료 계정/학생 계정은 불가 |
| **Contributor 이상 권한** | 쿼터 신청 및 VM 배포에 필요 |
| **vCPU 쿼터** | NDASv4 Family 기준 VM당 96 vCPU. 32 VMs = 3,072 vCPU 필요 |
| **리전 내 물리 용량** | 쿼터가 있어도 실제 물리 용량이 없으면 배포 실패 |
| **구독 유형 제한** | 스폰서드 계정, 무료 평가판 불가 |

> ⚠️ **중요:** ND 시리즈는 On-demand Capacity Reservation **미지원** (공식 문서 명시)
> → 용량 보장 방법은 AE/STU 경유 내부 Capacity Pre-approval 뿐임

## 3. 규모별 수급 경로 (규모에 따라 절차 자체가 다름)

### 규모 기준표 (ND96asr_v4 기준, GPU 8개/VM)

| 규모 | GPU 수 | VM 수 | vCPU 수 |
|---|---|---|---|
| 소규모 | ~64 GPUs | ~8 VMs | ~768 vCPU |
| 중규모 | ~256 GPUs | ~32 VMs | ~3,072 vCPU ← **이 케이스** |
| 대규모 | ~1,000 GPUs | ~125 VMs | ~12,000 vCPU |
| 초대규모 | 1,000+ GPUs | 125+ VMs | 12,000+ vCPU |

---

### 소규모 (~64 GPUs / ~8 VMs)

**셀프서비스 가능**

1. **리전 가용성 확인**
   ```bash
   az vm list-skus --location <region> --query "[?name=='Standard_ND96asr_v4']" -o table
   ```
2. **Portal 쿼터 신청:** Azure Portal → Quotas → NDASv4 Family vCPUs → New Quota Request
   - 자동 승인되는 경우 많음 (수 분 내)
3. **VM 배포:** PAYG로 즉시 배포

---

### 중규모 (~256 GPUs / ~32 VMs) ← 이 케이스

**쿼터 + 리전 용량 확인 필요**

1. **리전 가용성 확인** (CLI 또는 VM 생성 화면에서 해당 SKU 선택 가능 여부)
2. **Portal 쿼터 신청:** 3,072 vCPU 요청
   - 자동 승인 가능하나 실패 시 Support Request로 연결
3. **VM 배포 시도:** 쿼터가 있어도 물리 용량 부족으로 실패할 수 있음
4. **배포 실패 시:** Azure Support 티켓 → "Capacity Request" 유형으로 특정 리전 용량 요청
5. **또는 AE/STU 경유:** 담당 영업에게 내부 Capacity Pre-approval 요청 (더 빠른 경우 있음)

> ND 시리즈는 On-demand Capacity Reservation **미지원** → 용량 보장 경로가 Support 티켓 또는 AE뿐

---

### 대규모 (~1,000 GPUs / ~125 VMs)

**영업 채널 필수, 셀프서비스 불가**

1. **담당 AE/STU에 사전 연락 필수**
   - 단일 리전에서 이 규모를 한번에 확보하는 것은 Microsoft 내부 Capacity Planning이 필요
2. **AE가 내부 Capacity Pre-approval 프로세스 진행**
   - 사전에 특정 리전·특정 날짜 기준으로 용량 잠금 요청
   - 보통 수일~수 주 소요
3. **쿼터 신청은 병행 진행** (Azure Support 티켓)
4. **배포 준비 완료 후 PAYG로 배포**

> 이 규모에서 단일 CSP로 확보가 어렵다고 고객이 판단한 것이 현실적 수치

---

### 초대규모 (1,000 GPUs 이상)

**Microsoft 본사 수준 협의 필요**

- AE → STU → Microsoft HQ 레벨의 대형 딜 협상
- **Azure Dedicated Host** 또는 **Azure Large Instances** 수준의 전용 인프라 계약 가능성
- 리드타임 수 개월 이상
- 단일 리전으로 조달 불가 → 복수 리전 분산 배포 또는 멀티 CSP 조합 불가피

## 5. 수급 경로 한눈에 보기

```
GPU 8~64개 (8VMs 이하)
  └─ Portal 쿼터 신청 → 자동 승인 → PAYG 배포

GPU ~256개 (32 VMs)  ← 이 케이스
  └─ Portal 쿼터 신청 → VM 배포 시도
       └─ 용량 부족 실패 시 → Azure Support Capacity Request
       └─ 또는 AE/STU 경유 Capacity Pre-approval

GPU ~1,000개 (125 VMs)
  └─ AE/STU 선접촉 필수 (셀프서비스 불가)
       └─ 내부 Capacity Pre-approval + 쿼터 신청 병행
       └─ 단일 CSP 확보 어려울 수 있음 → 멀티 CSP 병행 고려

GPU 1,000개 초과
  └─ Microsoft 본사급 대형 딜 협상
       └─ 수 개월 리드타임, 복수 리전 필요
```

---

## 5. 참고: ND 시리즈 Capacity Reservation 미지원

Azure On-demand Capacity Reservation은 ND 시리즈를 **명시적으로 미지원**합니다.

> 지원되지 않는 GPU 시리즈: ND-series, NCads H100 v5, HB-series, HC-series
> 출처: [On-demand capacity reservation 공식 문서](https://learn.microsoft.com/en-us/azure/virtual-machines/capacity-reservation-overview)

→ ND 시리즈는 용량 사전 예약이 불가하므로, 대규모 배포 시 반드시 **AE/STU 경유 사전 협의**가 필요합니다.
