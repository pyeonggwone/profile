## 1. GPU 확보 전제 조건

ND 시리즈 GPU를 Azure에서 확보하려면 아래 조건이 모두 충족되어야 합니다.

| 조건 | 내용 |
|---|---|
| **유효한 Azure 구독** | EA, CSP, PAYG 모두 가능. 무료 계정/학생 계정은 불가 |
| **Contributor 이상 권한** | 쿼터 신청 및 VM 배포에 필요 |
| **vCPU 쿼터** | NDASv4 Family 기준 VM당 96 vCPU. 32 VMs = 3,072 vCPU 필요 |
| **리전 내 물리 용량** | 쿼터가 있어도 실제 물리 용량이 없으면 배포 실패 |

> ⚠️ **중요:** ND 시리즈는 On-demand Capacity Reservation **미지원** (공식 문서 명시)

### 중규모 (~256 GPUs / ~32 VMs) ← 이 케이스

**쿼터 + 리전 용량 확인 필요**

#### STEP 1. 리전 가용성 확인

- **방법 A - CLI:**
  ```bash
  az vm list-skus --location <region> --query "[?name=='Standard_ND96asr_v4']" -o table
  ```
  결과에 `NotAvailableForSubscription` 제한이 있으면 해당 리전에서 배포 불가

  Azure Portal → Virtual Machines → Create → Region 선택 → Size 검색 (`ND96`) → SKU가 목록에 표시되면 가용 가능

  → 구독 선택 → Provider: `Compute` → 리전 선택 → `NDASv4 Family vCPUs` 검색 → **New Quota Request**
- **요청 수량:** `3,072` vCPU (32 VMs × 96 vCPU)
- **처리 시간:** 자동 승인 시 수 분 ~ 수 시간 / 수동 검토 시 1~3 영업일
- **필요 정보:**
  | 항목 | 값 |
  |---|---|
  | Subscription ID | (Acryl AI 구독 ID, 클루커스 보유) |
  | Region | Sweden Central 등 배포 대상 리전 |
  | VM Family | NDASv4 Family vCPUs |
  | Requested Limit | 3,072 |
  | Justification | AI training workload, 1,000 GPU cluster test, 1–4 weeks duration |

#### STEP 3. VM 배포 시도 (쿼터 승인 후)

- 배포 방법: Portal 또는 CLI (`az vm create ...`)
- ⚠️ 쿼터가 있어도 물리 용량 부족이면 배포 실패 가능 → STEP 4로

#### STEP 4-A. 배포 실패 시 — Azure Support Capacity Request

- **신청 주체:** 클루커스 (구독 Owner/Contributor 권한으로 티켓 생성) 또는 Acryl AI 직접
- **신청 URL:**
  `https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest`
  → Issue Type: `Service and subscription limits (quotas)` → `Compute-VM`
  → 또는 Issue Type: `Technical` → 검색 `Capacity` → `VM Capacity Reservation`
- **담당:** Microsoft Azure Support 팀 (자동 배정, 직접 선택 불가)
- **필요 정보:**
  | 항목 | 값 |
  |---|---|
  | Subscription ID | (Acryl AI 구독 ID) |
  | VM Size | Standard_ND96asr_v4 |
  | VM Count | 32 |
  | Region (우선순위 포함) | Sweden Central > East US > West Europe |
  | Required Start Date | (배포 예정일) |

  | Use Case | Large-scale GPU cluster for AI training/inference |
- **처리 시간:** 보통 3~7 영업일, 빠르면 1~2일

#### STEP 4-B. 또는 영업 채널 경유 (STEP 4-A 병행 또는 대체)

> **현재 계약 구조:** Acryl AI ← CSP: 클루커스 ← Microsoft

**경로 A — 클루커스 → Microsoft PDM (메인 경로)**
- 클루커스가 담당 Microsoft PDM(Partner Development Manager)에게 Capacity Request 이메일 요청
- PDM 확인: 클루커스가 Partner Center 로그인 → 화면 우측 상단 계정 메뉴에서 담당자 확인
  `https://partner.microsoft.com/en-US/dashboard/`

  `https://partner.microsoft.com/en-US/support`

**경로 B — Acryl AI에 직접 Microsoft AE 있는 경우** (EA 또는 별도 계약 보유 시)
- Azure Portal → Cost Management + Billing → 구독 선택 → **Properties**에서 Account Executive 이름/이메일 확인 후 직접 연락
  `https://portal.azure.com/#view/Microsoft_Azure_CostManagement/Menu/~/properties`

- **요청 내용:** 내부 Capacity Pre-approval 요청 (이메일로)
- **필요 정보:** STEP 4-A와 동일 + Company: Acryl AI, CSP Partner: 클루커스, TPID (클루커스 보유)
- **장점:** Support 티켓 대비 처리 빠른 경우 있음 (내부 채널 직접 활용)

> ND 시리즈는 On-demand Capacity Reservation **미지원** → 용량 보장 경로가 Support 티켓 또는 AE뿐

## 5. 참고: ND 시리즈 Capacity Reservation 미지원

Azure On-demand Capacity Reservation은 ND 시리즈를 **명시적으로 미지원**합니다.

> 지원되지 않는 GPU 시리즈: ND-series, NCads H100 v5, HB-series, HC-series
> 출처: [On-demand capacity reservation 공식 문서](https://learn.microsoft.com/en-us/azure/virtual-machines/capacity-reservation-overview)

→ ND 시리즈는 용량 사전 예약이 불가하므로, 대규모 배포 시 반드시 **AE/STU 경유 사전 협의**가 필요합니다.

- AE가 MSFT 내부 시스템(외부 직접 접근 불가)을 통해 특정 리전·날짜 기준 용량 잠금 요청
- **처리 시간:** 수일 ~ 수 주 (규모 및 리전 가용성에 따라 상이)
- 승인 시 AE가 고객/PTC에 결과 및 배포 가능 시점 통보

---

#### STEP 3. 쿼터 신청 병행 (STEP 2와 동시 진행)

- **신청 위치:** [Azure Portal → Quotas](https://portal.azure.com/#view/Microsoft_Azure_Capacity/QuotaMenuBlade/~/myQuotas)
- 또는 [Azure Support 새 요청](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest) → `Service and subscription limits (quotas)`
- **요청 수량:** `12,000` vCPU (125 VMs × 96 vCPU)
- > Capacity Pre-approval 없이 쿼터만으로는 배포 불가하지만, 쿼터가 없으면 Capacity 승인 후에도 배포 안 됨 → 반드시 병행

---

#### STEP 4. Capacity 확보 확인 후 PAYG 배포

- AE로부터 Capacity 확보 확인 통보 수령 후 배포 진행
- PAYG 또는 AE와 협의된 요금제로 배포
