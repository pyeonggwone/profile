## 경로 1 — 리전 가용성 사전 확인 (배포 전 필수)

**실행 주체:** 클루커스 또는 PTC (내부 구독 사용)

### CLI 명령어

```bash
az vm list-skus --location canadacentral --query "[?name=='Standard_ND96asr_v4']" -o table
az vm list-skus --location eastus         --query "[?name=='Standard_ND96asr_v4']" -o table
az vm list-skus --location westus2        --query "[?name=='Standard_ND96asr_v4']" -o table
az vm list-skus --location swedencentral  --query "[?name=='Standard_ND96asr_v4']" -o table
az vm list-skus --location koreacentral   --query "[?name=='Standard_ND96asr_v4']" -o table
```

### 결과 해석

| Restrictions 컬럼 | 의미 | 다음 액션 |
|---|---|---|
| 비어 있음 | 배포 가능 | 경로 2로 이동 |
| `NotAvailableForSubscription` | 해당 구독/리전 조합 불가 | 경로 3으로 바로 이동 |
| `Location` | 리전 자체 미지원 | 다음 리전 시도 |

### Portal 대안

```
Azure Portal → Virtual Machines → Create
→ Region 선택 → Size 검색창에 "ND96" 입력
→ SKU 표시 여부 확인
```

## 경로 2 — Azure Portal 쿼터 신청 (셀프서비스)

**실행 주체:** 클루커스 (Acryl AI 구독 Contributor 이상 권한 필요)

**URL:**
```
https://portal.azure.com/#view/Microsoft_Azure_Capacity/QuotaMenuBlade/~/myQuotas
```

### 화면 입력 순서

1. 위 URL 접속 → 구독 선택 (Acryl AI 구독)
2. `Provider` → `Compute`
3. `Region` → `Canada Central`
4. 검색창 → `NDASv4` 입력
5. `NDASv4 Family vCPUs` 행 우측 연필 아이콘 클릭

### 팝업 입력값

| 필드 | 입력값 |
|---|---|
| New limit | `3072` |
| Justification | 아래 텍스트 복사 붙여넣기 |

**Justification 텍스트:**
```
Requesting 32x Standard_ND96asr_v4 VMs (256 A100 GPUs) for InfiniBand-connected
HPC cluster. Customer: Acryl AI / CSP: Cloukers.
Duration: 1–4 weeks.
Use case: Distributed multi-GPU orchestration validation for "Jonathan" MLOps platform.
Customer previously deployed identical SKU in Canada Central in March 2026
via Azure Support ticket.
Committed spend estimate: $148K (1 week) ~ $635K (1 month). PO confirmed ($600K).
```

### 처리 결과

| 결과 | 소요 시간 | 다음 액션 |
|---|---|---|
| 자동 승인 | 수 분~수 시간 | 경로 2-B (VM 배포 시도) |
| 수동 검토 | 1~3 영업일 | 대기 후 확인 |
| 거절 / Backlog | — | 경로 3으로 이동 |

### 2-B. 쿼터 승인 후 VM 배포 시도

쿼터가 있어도 물리 용량 부족 시 배포 실패 가능 → 실패 시 경로 3으로 이동

Required start date: [배포 예정일 입력]

## 경로 6 — AE/STU → DC GTM Alias 에스컬레이션 (최후 수단)

**실행 주체:** AE 또는 STU (Microsoft 내부)

**조건:** UAT로도 해결 안 될 때

### 에스컬레이션 흐름

```
AE/STU
 → MSX Milestone 업데이트 (경로 5)
 → UAT 생성 (경로 5)
 → DC GTM Alias (지역별 라우팅)
      Canada Central = Americas 권역
      → DC GTM Americas alias
 → Compute Capacity Team (내부 triage)
 → Capacity Pre-approval
```

### AE/STU 찾는 방법

- **AE:** TPD 김민지 또는 CSAM에게 문의 → *"Acryl AI 담당 AE 누구야?"*
- **PDM:** 클루커스가 아래 URL에서 직접 확인
  ```
  https://partner.microsoft.com/en-US/dashboard/
  → 우측 상단 계정 메뉴 → 담당 PDM 이름·이메일 확인
  ```
