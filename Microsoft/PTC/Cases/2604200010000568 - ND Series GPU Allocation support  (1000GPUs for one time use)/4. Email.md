### 5. 이 케이스(256 GPUs / 32 VMs) 신청 절차 — 실제 화면 입력값 포함

#### STEP 1. 리전 가용성 확인 (배포 전 반드시 실행)

**CLI** (Acryl AI 구독에 로그인 후):

```bash
az vm list-skus --location canadacentral --query "[?name=='Standard_ND96asr_v4']" -o table
az vm list-skus --location eastus         --query "[?name=='Standard_ND96asr_v4']" -o table
az vm list-skus --location westus2        --query "[?name=='Standard_ND96asr_v4']" -o table
az vm list-skus --location swedencentral  --query "[?name=='Standard_ND96asr_v4']" -o table
az vm list-skus --location koreacentral   --query "[?name=='Standard_ND96asr_v4']" -o table
```

- 출력 `Restrictions` 컬럼에 `NotAvailableForSubscription` 또는 `Location` 표시 시 → 해당 리전·구독 조합으로 배포 불가 (쿼터/Support 신청 필요).
- `Restrictions` 가 비어 있으면 → 쿼터 신청 후 즉시 배포 시도 가능.

**Portal 대안:**
Azure Portal → Virtual Machines → Create → Region 선택 → Size 검색창에 `ND96` → SKU 표시 여부 확인.

---

#### STEP 2. Azure Portal Quotas — 쿼터 신청

**신청 주체:** 클루커스 (Acryl AI 구독 Contributor 이상 권한 필요) 또는 Acryl AI 직접

**URL:** `https://portal.azure.com/#view/Microsoft_Azure_Capacity/QuotaMenuBlade/~/myQuotas`

**화면 진입 순서:**
1. 위 URL 접속 → 좌측 상단 구독 선택 (Acryl AI 구독)
2. `Provider` 드롭다운 → `Compute` 선택
3. `Region` 드롭다운 → `Canada Central` 선택 (1순위)
4. 검색창에 `NDASv4` 입력 → `NDASv4 Family vCPUs` 행 확인
5. 해당 행 우측 연필 아이콘 클릭 → **New Quota Request** 팝업 열림

**팝업 입력값:**

| 화면 필드 | 입력값 |
|---|---|
| New limit | `3072` |
| Justification | `Large-scale AI training workload. Requesting 32x Standard_ND96asr_v4 VMs (256 A100 GPUs) for InfiniBand-connected HPC cluster. Customer: Acryl AI. CSP partner: Cloukers. Expected duration: 1–4 weeks. Workload: distributed model training for "Jonathan" MLOps platform multi-GPU orchestration validation. Customer previously deployed identical SKU in Canada Central in March 2026 via Azure Support ticket.` |

- 처리 시간: 자동 승인 시 수 분~수 시간 / 수동 검토 시 1~3 영업일.
- Justification이 구체적일수록 자동 거절 없이 수동 검토로 넘어가는 확률이 높아집니다.

**쿼터가 있어도 물리 용량 부족이면 배포 실패** 가능 → 실패 시 STEP 4 진행.

#### STEP 4-A. Azure Support — Capacity Request 티켓 (← 고객 2026-03 사용 경로)

**고객사가 캐나다 리전에서 ND 시리즈를 확보한 실제 경로입니다. 동일 경로 우선 권장.**

**URL:** `https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest`

**화면 입력 순서:**

| 단계 | 필드 | 입력값 |
|---|---|---|
| 1 | Issue type | `Service and subscription limits (quotas)` |
| 2 | Subscription | Acryl AI 구독 선택 |
| 3 | Quota type | `Compute-VM (cores-vCPUs) subscription limit increases` |
| 4 | Problem details | (클릭 후 세부 입력) |
| 5 | Deployment model | `Resource Manager` |
| 6 | Location | `Canada Central` (1순위) |
| 7 | Quota type (세부) | `Standard NDASv4 Family vCPUs` |
| 8 | New limit | `3072` |

**Additional details (자유 입력란):**

```
Customer: Acryl AI
Partner CSP: Cloukers (클루커스)

Requested VM count: 32 (256 GPUs)
```

> 참고: Issue Type을 `Technical` → 검색 `Capacity` → `VM Capacity Reservation` 으로 여는 경로도 있으나, 위 `Service and subscription limits (quotas)` 경로가 일반적이며 고객 3월 사용 경로와 동일합니다.

### 6. 1,000 GPUs (125 VMs) 단일 CSP 확보 — 참고 정보

고객사도 단일 CSP에서 1,000장 확보가 어렵다고 판단해 AWS/GCP 병행 예정으로 안내해 주셨습니다.

만약 Microsoft 단독으로 1,000 GPU 시도를 검토하실 경우 절차는 아래와 같습니다 (셀프서비스 불가):

1. **PDM/AE 선접촉 필수** (클루커스 → MS PDM, 또는 Acryl AI → MS AE)
2. AE → Microsoft STU → Regional Capacity Team 내부 Pre-approval (수일 ~ 수 주)
3. 쿼터 신청 병행 (`12,000` vCPU = 125 VMs × 96 vCPU)
4. Capacity 확보 통보 수령 후 PAYG 배포

**핵심 제약:**
- ND 시리즈는 **On-demand Capacity Reservation 미지원** (공식 문서)
  → https://learn.microsoft.com/en-us/azure/virtual-machines/capacity-reservation-overview
- 단일 리전 1,000 GPU 확보는 리드타임 수 주 이상, 리전 분산 또는 멀티 CSP 조합이 현실적.
