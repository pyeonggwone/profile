## 신규 Work Item 생성

1. 위 URL 접속 후 Microsoft 계정으로 로그인
2. 좌측 메뉴 → `Boards` → `Work Items` 이동
3. 우측 상단 `+ New Work Item` 클릭 → `ACTIONS` 선택
4. `NEW ACTIONS` 화면이 열리면 먼저 `Title` 입력
   - 예시: `[Capacity][Acryl AI] ND96asr_v4 1,000 GPU quota request - Canada Central`
   - `Title`은 필수값이며, 비워두면 `Field 'Title' cannot be empty.` 오류로 저장 불가
5. 기본 상태값 확인
   - `State`: `In Progress`
   - `Reason`: `Moved to state In Progress`
   - `Area`: `Unified Action Tracker`
   - `Iteration`: `Unified Action Tracker`
6. 아래 섹션을 순서대로 입력
   - `Details`: Description, Customer Scenario and Desired Outcome, Customer Impact, Alternative Design Options, Status Update
   - `Action Info`: Action Category, Requested Date, Action Priority, Due Date, FY, Quarter
   - `Non AI Capacity`: Request Type, Subscription ID, Workload Environment, Regional/Zonal, SKU, UOM, Quantity, SR#
   - `Milestone Details`: Milestone Status, Milestone Reason, Help Needed, AzureCapacityType, AzurePreferredRegion
   - `Contact`: Requestors, Managing Owner, AE Alias
   - `Opportunity Details`: Est. Monthly Usages, BACV, Customer Commitment
   - `Account Info`: Account, Country, Segment
7. 입력 완료 후 우측 상단 `Save` 클릭
8. 저장 후 생성된 Work Item 번호를 기록하고 TPD 김민지에게 공유

> 화면에 `Link to UAT V2` 안내가 표시되면, 신규 UAT 포털에서도 동일 Action을 열 수 있다. 다만 Work Item 생성과 저장은 현재 `NEW ACTIONS` 화면에서 먼저 완료한다.

### Priority 검토

이 케이스는 고객 계약 및 매출 영향이 크고 GPU capacity가 PO 진행의 주요 blocker이지만, 운영 중인 production 장애나 서비스 중단 상황은 아니다. 따라서 `1 - Critical`보다는 `2 - High`로 등록하는 것을 권고한다.

- `1 - Critical`: production outage, 심각한 서비스 중단, 즉시 임원 escalation이 필요한 장애성 이슈에 적합
- `2 - High`: 확정된 PO/매출 기회가 있고 일정 지연 리스크가 큰 capacity blocker에 적합

## 신청 시 입력 필요 여부 요약

전부 다 채울 필요는 없다. UAT 화면에는 여러 프로그램 공통 필드가 함께 표시되므로, 이 케이스에서는 **capacity triage에 필요한 필드**와 **계약/고객/기회 식별에 필요한 필드**만 우선 입력한다.

| 구분 | 처리 방식 | 대상 필드 |
|---|---|---|
| 필수 입력 | 저장 또는 triage 라우팅을 위해 입력 | `Title`, `Description`, `Customer Scenario and Desired Outcome`, `Customer Impact`, `Action Category`, `Requested Date`, `Action Priority`, `Due Date`, `Request Type`, `Subscription ID`, `Workload Environment`, `Regional/Zonal`, `SKU/Product/Plan`, `UOM`, `Quantity`, `Support Request #(SR)`, `Requestors`, `Managing Owner`, `Account`, `Country`, `Segment`, `Milestone Status`, `Milestone Reason`, `Help Needed`, `AzureCapacityType`, `AzurePreferredRegion` |
| 권장 입력 | capacity 팀/AE/TPD가 비즈니스 영향을 판단하기 쉽도록 입력 | `Estimated Monthly Usage ($)`, `Total Estimated Monthly Usage (USD)`, `Est. Monthly Usages (USD)`, `BACV`, `Customer Commitment`, `Opportunity ID`, `Opportunity Name`, `Opportunity Link`, `AE Alias`, `Status Update`, `Alternative Design Options` |
| 확인 후 입력 | MSX/CRM 또는 TPD/AE 확인 필요 | `TPID`, `Account ID`, `Opportunity Stage`, `Opportunity Size`, `Solution Area`, `Partner One Name`, `Milestone ID`, `Milestone Link`, `Milestone Owner`, `CE Alias`, `SSSP Alias` |
| 공란 또는 기본값 유지 | 현재 케이스와 직접 관련 없거나 종료 시점에만 입력 | `Closure Statement`, `Active Action Resolution`, `Email Automation`, `Federal`, `Feature Parity Request`, `MACC`, `RAID`, `Meeting`, `Product Info`, `Support Agreement`, `Scans`, `Technical Feedback` 계열 |
