# research_notes

summary: The customer needs architectural guidance for consolidating multiple affiliated companies into one Azure tenant while preserving separate company domains and operational identity boundaries.

facts:
- Microsoft Entra ID supports adding and verifying multiple custom domains in one tenant, but a custom domain can be verified in only one tenant at a time.
- Tenant-level controls such as Conditional Access, identity protection, audit, directory roles, and some governance settings may affect all entities unless carefully scoped.
- Administrative units, role assignments, groups, naming standards, and lifecycle governance can help delegate administration, but they do not create fully separate tenant boundaries.
- Existing tenants require discovery of verified domains, subscriptions, identities, applications, mail flow, and policy dependencies before consolidation decisions.
- Multi-tenant organization and cross-tenant synchronization can be considered when full consolidation is not immediately feasible or when phased coexistence is required.

follow_up_questions:
- How many current Azure/Microsoft Entra tenants, subscriptions, verified domains, and users are in scope for each affiliated company?
- Which identity source is used today for each company: cloud-only, Active Directory, Microsoft Entra Connect, Cloud Sync, or federation?
- Are Exchange Online, Teams, SharePoint, or third-party applications also included in the consolidation initiative?
- What administrative separation, audit, data residency, compliance, and security policy requirements must remain company-specific?
- Is the target model full tenant consolidation, phased coexistence, or long-term multi-tenant operation with collaboration controls?
