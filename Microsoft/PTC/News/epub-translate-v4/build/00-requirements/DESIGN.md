# 요구사항 실행 기준

## 목적

`README.md`의 내용을 실제 구현 판단 기준으로 고정한다. 이 문서는 모든 실행계획의 상위 요구사항이다.

## 핵심 요구사항

- `input/`에 있는 파일을 자동 스캔한다.
- 지원 포맷은 `epub`, `azw3`, `mobi`, `kfx`다.
- 파일별 실제 포맷을 감지한 뒤 포맷별 native adapter로 분기한다.
- 번역 후 `output/`에 원본과 같은 포맷으로 저장한다.
- 책별 메타데이터와 번역 사용량은 `ebook-metadata/`에 JSON으로 저장한다.
- DRM 감지 파일은 번역하지 않고 skip 상태로 기록한다.
- 초기 MVP는 완전한 보존보다 성공적인 번역과 저장 확인을 우선한다.
- 장기 목표는 포맷별 구조, 레이아웃, 리소스, 메타데이터 보존이다.

## 구현하지 않을 것

- `epup-translate-v3` 직접 수정
- 포맷 변환 기반 처리로 요구사항 대체
- DRM 우회 또는 제거
- KFX 완성형 구현을 MVP 완료 조건으로 설정
- 외부 SaaS 기반 전자책 변환 API 의존

## 성공 기준

- 각 포맷의 처리 경계가 명확하다.
- 공통 번역 엔진이 포맷 세부 구조를 알지 않는다.
- 포맷별 reader/writer 책임이 분리되어 있다.
- 실패한 책도 metadata JSON으로 추적된다.
