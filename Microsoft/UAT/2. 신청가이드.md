# AI Infra GPU UAT 신청 가이드

## 1. 신청 링크

### 신청 시작 위치

AI Infra (3P GPU) capacity 요청은 UAT portal에서 직접 생성하지 말고, **MSX milestone view**에서 시작한다.

| 목적 | 링크 또는 위치 | 사용 방법 |
|---|---|---|
| UAT 신청 | MSX Opportunity > Milestone view > **Add UAT Action** | GPU capacity 요청은 반드시 이 경로로 생성 |
| UAT Portal | https://uatracker.microsoft.com | 제출 후 action 조회 및 상태 확인용 |
| UAT Shortcut | https://aka.ms/uat | UAT portal 접근용 |
| AI UAT Case Status | https://aka.ms/aiuatcases | AI Capacity team에 배정된 UAT 상태 확인 |
| Product Availability by Region | 확인필요 | VM SKU가 region에 실제 제공되는지 확인 |
| Azure VM ND A100 v4 문서 | https://learn.microsoft.com/ko-kr/azure/virtual-machines/sizes/gpu-accelerated/ndasra100v4-series?tabs=sizebasic | ND A100 v4 및 Infiniband 지원 확인 |

### 신청 전 필수 확인

MSX에서 아래 값이 설정되어 있지 않으면 올바른 AI Capacity queue로 들어가지 않을 수 있다.

| MSX 항목 | 입력값 |
|---|---|
| Milestone workload | AI: AI Infra (GPUs) |
| Milestone status | Blocked |
| Milestone Status reason | Capacity/Service Availability |

위 값은 Opportunity owner가 수정해야 한다. 수정 권한이 없으면 **Opportunity owner 확인필요**로 표시하고 owner에게 변경을 요청한다.

## 2. 신청 단위

UAT는 **one UAT per ask** 기준으로 제출한다.

| 요청 조건 | 제출 방식 |
|---|---|
| A100과 H100을 모두 요청 | A100 1건, H100 1건으로 분리 제출 |
| region이 다름 | region별로 분리 제출 |
| subscription이 다름 | subscription별로 분리 제출 |
| 같은 SKU, 같은 region, 같은 subscription | 하나의 UAT로 제출 가능 |

현재 케이스 기준 신청 단위:

| 구분 | 신청 제목 | 수량 |
|---|---|---:|
| A100 | [Capacity] A100 256 GPU Capacity Request | 256 GPUs |
| H100 | [Capacity] H100 256 GPU Capacity Request | 256 GPUs |

## 3. 항목별 채울 내용

### 공통 필드

| UAT 항목 | 입력 내용 | 모를 경우 |
|---|---|---|
| Request Type | Field Request | 확인필요 |
| Domain / Workload | AI Infra (3P GPU) 또는 AI: AI Infra (GPUs) | MSX milestone workload 확인필요 |
| Assign to Corp | True / Yes | 반드시 True로 설정 |
| Feedback | No | 제품 기능 개선 요청이면 확인필요 |
| Requestor | 본인 이름 또는 자동 입력값 유지 | 확인필요 |
| Action Owner | Azure AI Capacity Triage 자동 지정 확인 | 자동 지정되지 않으면 queue 설정 확인필요 |
| Assigned Team | Azure AI Capacity Triage 또는 AI Capacity team | 자동 지정되지 않으면 확인필요 |
| Priority | High 또는 고객 영향 기준 선택 | 고객 deadline/impact 확인필요 |
| Due Date | 고객이 capacity 확보를 필요로 하는 날짜 | 고객 일정 확인필요 |
| Customer / Account | 고객사명 입력 | 정확한 account 확인필요 |
| TPID | 고객 TPID 입력 | TPID 확인필요 |
| Opportunity ID | MSX Opportunity ID 입력 또는 자동 연계 확인 | Opportunity owner에게 확인필요 |
| Milestone ID | Blocked milestone ID 입력 또는 자동 연계 확인 | MSX milestone 확인필요 |
| Milestone Status | Blocked | MSX에서 먼저 수정 필요 |
| Milestone Status Reason | Capacity/Service Availability | MSX에서 먼저 수정 필요 |

### AI Infra / 3P GPU 섹션

| UAT 항목 | 입력 내용 | 모를 경우 |
|---|---|---|
| VM Series / SKU | A100 요청: ND A100 v4 series / H100 요청: ND H100 v5 series | Product Availability by Region에서 확인필요 |
| GPU Quantity | 256 GPUs | 고객 최종 요청 수량 확인필요 |
| Requested Region | SKU가 실제 제공되고 고객 요구에 맞는 region만 입력 | Product Availability by Region 확인필요 |
| Subscription ID | 배포 대상 Azure subscription ID | 고객 또는 account team 확인필요 |
| Use Case | Infiniband 기반 대규모 AI training workload | 고객 workload 상세 확인필요 |
| Workload Detail | 모델 학습, autonomous driving model training 등 실제 사용 목적 입력 | 고객 기술 담당자 확인필요 |
| Deployment Timeline | capacity가 필요한 목표 일정 | 고객 일정 확인필요 |
| Commitment Type | Reserved Instance 1년 이상 가능 여부 기재 | 고객 구매 의사 확인필요 |
| Payment Model | PayGo는 현재 지원되지 않음. RI 가능 여부 중심으로 작성 | Deal Desk 확인필요 |
| Additional Comments | 경쟁사 확보 상황, region 제약, Infiniband 필요성, 기타 배경 입력 | 모르는 내용은 확인필요로 남김 |

### Description 입력문

아래 문장을 기본으로 사용하고, SKU와 region만 신청 건에 맞게 수정한다.

```text
Requesting Azure AI GPU capacity for customer AI training workload.

The customer requires Infiniband-enabled GPU capacity. The primary requirement is not limited to a specific GPU model, but the target SKU for this request is [ND A100 v4 / ND H100 v5].

Requested quantity: 256 GPUs
Requested region: 확인필요
Target subscription: 확인필요
Required timeline: 확인필요
Commitment: 확인필요 - Reserved Instance 1 year or above should be validated with customer
```

### Customer Scenario & Desired Outcome 입력문

```text
The customer is planning to run large-scale AI workloads that require Infiniband-enabled GPU infrastructure on Azure. The desired outcome is to secure deployable GPU capacity for the requested SKU and region so the customer can proceed with the AI workload deployment on Azure.

The customer can consider either A100 or H100 class GPU capacity as long as the SKU supports Infiniband and meets the workload requirement.
```

### Customer Impact 입력문

```text
The customer is evaluating multi-cloud options for GPU capacity. AWS has already provided 256 GPUs, and the customer is also checking capacity with other cloud providers such as GCP.

If Microsoft cannot provide feasible GPU capacity, the workload may move to another cloud provider. Securing capacity is important for Azure adoption and for retaining the AI workload opportunity.

Business impact details such as revenue amount, project deadline, and production go-live date are 확인필요.
```

### Workaround Details 입력문

```text
The customer is evaluating alternative GPU capacity options across multiple cloud providers. AWS has already provided 256 GPUs.

Previous Azure support quota request was tracked under SR#2604270030000567, but UAT GPU capacity escalation should be handled through the AI Infra GPU UAT process and should not rely on the SR number as the UAT routing mechanism.

No confirmed Azure workaround is available yet. Region and SKU availability are still under validation.
```

### Additional Comments 입력문

```text
Important notes:
- The key customer requirement is Infiniband-enabled GPU capacity.
- The customer is not strictly limited to A100 if H100 capacity is available and meets the workload requirement.
- Product Availability by Region must be checked before confirming requested regions.
- Canada Central and East US were previously indicated as not approvable by Azure Capacity Customer Experience.
- West US 2, Sweden Central, and West Europe were under review, but SKU availability must be revalidated.
- Sweden Central was observed not to show ND A100 v4 in Product Availability by Region.
- Previous Canada Central usage was NC24rs v3, not ND A100.
```

## 6. 모르면 확인필요로 둘 항목

아래 항목은 임의로 작성하지 않는다. 모르면 **확인필요**로 남기고 customer, Opportunity owner, Deal Desk, AI GBB, 또는 account team에 확인한다.

| 항목 | 확인 대상 |
|---|---|
| Customer account 정확한 명칭 | Account team / MSX |
| TPID | Account team / MSX |
| Opportunity ID | Opportunity owner |
| Milestone ID | Opportunity owner |
| Subscription ID | Customer / Account team |
| Requested Region | Product Availability by Region / Customer requirement |
| 실제 SKU availability | Product Availability by Region |
| Required timeline | Customer |
| Revenue impact 또는 deal size | Account team |
| RI 1년 이상 가능 여부 | Customer / Deal Desk |
| AI GBB validation 필요 여부 | AI GBB |
| Deal Desk validation 필요 여부 | Deal Desk |

## 7. 제출 전 체크리스트

- [ ] MSX milestone workload가 **AI: AI Infra (GPUs)** 인지 확인
- [ ] MSX milestone status가 **Blocked** 인지 확인
- [ ] MSX milestone status reason이 **Capacity/Service Availability** 인지 확인
- [ ] MSX milestone view에서 **Add UAT Action**으로 시작했는지 확인
- [ ] A100과 H100을 별도 UAT로 분리했는지 확인
- [ ] Requested Region에 해당 SKU가 실제 존재하는지 확인
- [ ] Assign to Corp가 **True** 인지 확인
- [ ] Use Case, Customer Scenario, Customer Impact가 비어 있지 않은지 확인
- [ ] 모르는 항목은 임의 입력하지 않고 **확인필요**로 남겼는지 확인
- [ ] SR#2604270030000567을 UAT routing 용도로 입력하지 않았는지 확인

## 8. 제출 후 확인

| 확인 항목 | 확인 방법 |
|---|---|
| Action ID 생성 여부 | UAT 제출 완료 화면 또는 UAT portal |
| Queue 배정 | Action Owner 또는 Assigned Team이 Azure AI Capacity Triage인지 확인 |
| 상태 확인 | https://aka.ms/aiuatcases |
| Stakeholder 공유 | 생성된 UAT link 공유 |
| 기존 SR 진행 | SR#2604270030000567 email thread에서 별도 추적 |

잘못된 queue로 들어간 경우 기존 action을 수정하는 방식으로 해결하지 말고, MSX milestone 설정을 바로잡은 뒤 새 UAT로 다시 제출한다.
