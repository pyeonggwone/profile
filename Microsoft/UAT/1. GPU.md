# AI Infra (3P GPU) UAT Escalation 정리

## 목적

본 문서는 Microsoft UAT에서 AI Infra (3P GPU) capacity 요청을 올릴 때 필요한 MSX 설정, UAT 작성 기준, 현재 케이스 진행 상태, 그리고 재신청 시 주의해야 할 사항을 정리한 문서다.

## 핵심 결론

- AI Infra (3P GPU) capacity 요청은 반드시 **MSX milestone view**에서 **Add UAT Action**으로 생성해야 한다.
- UAT portal 또는 DevOps 경로에서 직접 생성한 action은 올바른 AI Capacity queue로 배정되지 않을 수 있다.
- Milestone workload는 반드시 **AI: AI Infra (GPUs)** 로 설정해야 한다.
- Milestone status는 **Blocked**, reason은 **Capacity/Service Availability** 로 설정해야 한다.
- ND series capacity 요청은 Corp level에서 처리되므로 **Assign to Corp = True** 여야 한다.
- A100과 H100은 서로 다른 모델이므로 각각 별도 UAT로 제출해야 한다.
- 현재 요청 규모는 1000 GPU가 아니라 **A100 256 GPU**, **H100 256 GPU** 기준으로 재정리한다.

## MSX 필수 설정

UAT action을 생성하기 전에 Opportunity owner가 MSX milestone을 다음과 같이 수정해야 한다.

| 항목 | 값 |
|---|---|
| Milestone workload | AI: AI Infra (GPUs) |
| Milestone status | Blocked |
| Milestone Status reason | Capacity/Service Availability |

이 설정을 완료한 뒤 MSX milestone view에서 **Add UAT Action**을 클릭해 action을 생성한다. 이 경로를 따르면 Action Owner가 자동으로 **Azure AI Capacity Triage**로 지정되어 AI Capacity team이 처리하는 올바른 UAT form과 queue로 들어간다.

## UAT 작성 기준

UAT에는 GPU 사용 목적과 workload를 구체적으로 작성해야 한다.

| 항목 | 작성 기준 |
|---|---|
| Use Case | 고객이 어떤 AI workload를 실행하려는지 설명 |
| VM Series | ND A100 v4, ND H100 v5 등 요청 SKU 명확히 기재 |
| Quantity | 필요한 GPU 수량 기재 |
| Region | 선호가 아니라 실제 요구 조건 기준으로 선택 |
| Additional Comments | VM 사용 목적, 모델 학습/추론 목적, 고객 제약 사항, 경쟁사 상황 등 보완 설명 |

Region은 가격 선호가 아니라 요구 조건 기준으로 선택해야 한다. 다른 region 가격과 맞춰야 하는 경우 Deal Desk에 문의한다.

## 지원 대상 SKU 및 예외

AI Infra UAT process는 다음 series capacity 요청을 지원한다.

- ND H100 series
- ND H200 series
- GB200 series

다음 series는 UAT 전에 Azure portal에서 support ticket을 먼저 생성하고, SCMTeam@microsoft.com 에 assistance를 요청해야 한다.

- NC series
- NG series
- NV series
- MI300x series
- ND A100 series

## Request 분리 기준

원칙적으로 **one UAT per ask** 기준으로 제출한다.

| 상황 | UAT 제출 방식 |
|---|---|
| 한 고객이 서로 다른 GPU model 2개 요청 | UAT 2개 제출 |
| 같은 고객이 A100과 H100 모두 요청 | A100 UAT 1개, H100 UAT 1개 제출 |
| subscription이 다름 | 별도 UAT 제출 |
| region이 다름 | 별도 UAT 제출 |

따라서 현재 고객 요청은 다음과 같이 분리한다.

| 요청 | 목표 수량 | 비고 |
|---|---:|---|
| A100 capacity | 256 GPU | 별도 UAT |
| H100 capacity | 256 GPU | 별도 UAT |

초기 논의에는 1000 GPU 요청이 있었으나 현실적으로 ND series 1000 GPU 확보 가능성이 낮고, 고객도 AWS/GCP 등 다른 cloud provider와 병행 논의 중이다. AWS에서는 이미 256 GPU를 공급받은 상태라고 공유되었다. 따라서 Microsoft 측 요청도 우선 256 GPU 단위로 제출하고 확보 가능한 수량을 고객에게 제안하는 방향이 현실적이다.

## 승인 및 우선순위 기준

- UAT 제출 자체가 capacity 확보 또는 승인 보장을 의미하지 않는다.
- AI Triage team이 case를 assessment한 뒤 next step을 안내한다.
- 대부분의 AI Infra request는 AI GBB 및 Deal Desk validation이 필요할 수 있다.
- 적용 가능한 경우 AI DWR triage로 넘어간다.
- Reserved Instances, 특히 1년 이상 commitment가 우선순위가 높다.
- PayGo deal은 현재 지원되지 않는다.
- DWR 또는 exemption process 승인 후 milestone을 **On Track**으로 변경할 수 있다.

## Status Tracking

AI Capacity team에 배정된 UAT escalation 상태는 다음 report에서 확인한다.

https://aka.ms/aiuatcases

기본적으로 open 상태의 **In Progress** request로 pre-filter되어 있으며, **State** dropdown을 변경하면 완료된 요청도 확인할 수 있다. Report는 daily refresh된다.

## 현재 케이스 상태

| 항목 | 내용 |
|---|---|
| Azure Support SR | SR#2604270030000567 |
| Support ticket 유형 | Azure Support ticket - Quota Increase |
| 현재 처리 경로 | Azure Capacity Customer Experience team과 SR email thread 기준 진행 |
| SR email thread | Re: 계산-VM(코어-vCPU) 구독 제한 증가에 대한 ... - TrackingID#2604270030000567 |

현재 Azure Support ticket 제출 후 SR 번호를 기준으로 UAT ticket을 신청한 상태였으나, 이후 확인 결과 ND series GPU capacity 요청은 UAT에서 올바른 queue로 들어가야 하며, SR 번호를 UAT에 넣는 방식은 별도 프로세스와 혼동될 수 있다는 피드백을 받았다.

## Region 검토 현황

Azure Capacity Customer Experience team에서 공유한 현재 region 검토 상태는 다음과 같다.

| Region | 상태 |
|---|---|
| Canada Central | 승인 불가 |
| East US | 승인 불가 |
| West US 2 | 검토 중 |
| Sweden Central | 검토 중 |
| West Europe | 검토 중 |

추가 확인 사항:

- Product Availability by Region 기준 Sweden Central에는 ND A100 v4가 없는 것으로 확인되었다.
- Canada Central에는 ND A100 v5가 없는 것으로 확인되었다.
- 과거 Canada Central 실사용 이력은 ND A100이 아니라 **NC24rs v3** 사용 이력이다.
- 원래 SKU가 없는 region을 UAT에 포함한 것이 가장 큰 오류로 지적되었다.

## 고객 요구사항

고객은 반드시 A100만 요구하는 것은 아니며, 핵심 요구는 **Infiniband가 지원되는 GPU SKU** 다.

확인된 사항:

- ND A100 v4 series는 Infiniband를 지원한다.
- 참고 문서: https://learn.microsoft.com/ko-kr/azure/virtual-machines/sizes/gpu-accelerated/ndasra100v4-series?tabs=sizebasic
- 고객은 A100 또는 H100 모두 검토 가능하다.
- H100의 경우 별도 UAT가 필요하다.
- 글로벌하게 ND H100 v5 물량이 부족하다는 피드백이 있었다.

## 기존 UAT 제출 내역

아래 action은 제출되었으나 잘못된 queue로 들어간 것으로 확인되었다.

| 요청 | UAT Action | 링크 | 문제 |
|---|---:|---|---|
| A100 256 GPU | 732048 | https://aka.ms/UA-Tracker-ManageAction?id=732048 | MSX workload 설정 및 queue routing 문제 |
| H100 256 GPU | 732049 | https://aka.ms/UA-Tracker-ManageAction?id=732049 | MSX workload 설정 및 queue routing 문제 |

기존 action은 UAT detail link에서 수정해 해결할 수 없으며, MSX milestone workload를 올바르게 설정한 뒤 다시 제출해야 한다는 피드백을 받았다.

## 잘못된 점 및 교정 포인트

| 문제 | 영향 | 교정 |
|---|---|---|
| UAT를 MSX milestone view에서 생성하지 않음 | AI Capacity queue로 routing되지 않음 | MSX에서 Add UAT Action으로 재생성 |
| Milestone workload가 AI: AI Infra (GPUs)가 아님 | Azure AI Capacity Triage로 자동 배정되지 않음 | Opportunity owner가 workload 수정 |
| Assign to Corp가 False였음 | Corp level capacity process로 들어가지 않음 | Assign to Corp = True |
| UAT 내용이 비어 있거나 view에 표시되지 않음 | Triage team이 요청 내용을 재확인해야 함 | Use Case, VM, quantity, region, business impact 상세 기재 |
| SKU가 없는 region 포함 | 요청 신뢰도 저하 및 처리 지연 | Product Availability by Region 기준으로 region 재검토 |
| SR 번호를 UAT에 넣으려 함 | SR process와 UAT process 혼동 | UAT에는 SR 번호를 넣지 않음 |

## 재제출 전 체크리스트

- [ ] Opportunity owner가 MSX milestone workload를 **AI: AI Infra (GPUs)** 로 변경
- [ ] Milestone status를 **Blocked** 로 설정
- [ ] Milestone Status reason을 **Capacity/Service Availability** 로 설정
- [ ] Product Availability by Region 기준으로 요청 region 재확인
- [ ] A100과 H100을 별도 ask로 분리
- [ ] 각 UAT의 GPU 수량을 256 GPU 기준으로 정리
- [ ] Use Case와 customer impact를 구체적으로 작성
- [ ] Requested region을 preference가 아니라 requirement 기준으로 작성
- [ ] Assign to Corp를 **True** 로 설정
- [ ] SR 번호는 UAT에 입력하지 않음
- [ ] 제출 후 Azure AI Capacity Triage 또는 AI Capacity queue로 배정되었는지 확인
- [ ] 제출된 UAT link를 stakeholder에게 공유

### A100 256 GPU

| 필드 | 작성 방향 |
|---|---|
| Title | [Capacity] A100 256 GPU Capacity Request |
| VM / SKU | ND A100 v4 series |
| Quantity | 256 GPUs |
| Region | Product Availability 확인 후 실제 지원 가능 region만 기재 |
| Use Case | Infiniband 기반 대규모 AI workload 학습 또는 실행 |
| Business Impact | 고객이 AWS/GCP와 병행 검토 중이며, AWS에서 이미 256 GPU를 확보한 상황. Microsoft capacity 확보 여부가 Azure 채택 및 workload 유치에 직접 영향 |
| Additional Comments | A100에 한정된 요구가 아니라 Infiniband 지원 GPU SKU 확보가 핵심 요구사항임 |
