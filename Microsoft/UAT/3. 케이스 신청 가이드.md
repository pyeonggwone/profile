# Acryl AI GPU Capacity UAT 신청 가이드

## 신청 경로

케이스는 **MSX Opportunity > Milestone view > Add UAT Action**에서만 생성한다.

UAT portal, DevOps, Azure support ticket, 기존 action 수정 경로로 새 GPU capacity request를 생성하지 않는다. 이번 케이스의 신청 경로는 하나다.

## MSX Milestone 설정

Add UAT Action을 누르기 전에 MSX milestone 값을 아래와 같이 맞춘다.

- **Milestone workload**: `AI: AI Infra (GPUs)`
- **Milestone status**: `Blocked`
- **Milestone Status reason**: `Capacity/Service Availability`

위 설정이 끝난 milestone view에서 **Add UAT Action**을 눌러 UAT를 생성한다.

GPU capacity 요청은 Corp level 처리가 필요하므로 **Assign to Corp**는 `True`로 설정한다.

SR 번호는 UAT routing 용도로 입력하지 않는다.

## 신청 단위

이번 케이스는 A100과 H100을 각각 별도 UAT로 신청한다.

- **A100 UAT**: `Standard_ND96asr_v4` 기준 `256 A100 GPUs`
- **H100 UAT**: `Standard_ND96isr_H100_v5` 기준 `256 H100 GPUs`

## 공통 입력값

A100 UAT와 H100 UAT 모두 아래 값을 동일하게 입력한다.

- **Customer**: `Acryl AI`
- **Partner CSP**: `Cloukers / 클루커스`
- **Request Type**: `GPU capacity / quota request`
- **Assign to Corp**: `True`
- **Duration**: `1 week, maximum 1 month`
- **Business Impact**: `$600K PO confirmed, GPU capacity is the main blocker`
- **Allocation Flexibility**: `Split-region allocation acceptable`
- **SR Number**: 입력하지 않음

## A100 UAT

### 기본 필드

- **Title**: `[Capacity][Acryl AI] A100 256 GPU capacity request`
- **Primary SKU**: `Standard_ND96asr_v4`
- **Alternative SKU**: `Standard_ND96amsr_A100_v4`
- **GPU Type**: `NVIDIA A100`
- **GPU Quantity**: `256 A100 GPUs`
- **VM Quantity**: `32 VMs`
- **vCPU Quantity**: `3,072 vCPU`
- **InfiniBand/RDMA**: `Supported`
- **Requested Region**: `East US, North Central US, South Central US, West US 2, UK South`

### Description

```text
Requesting Azure AI GPU capacity for Acryl AI via Cloukers / 클루커스.

The customer requires InfiniBand-enabled A100 GPU capacity for large-scale distributed multi-GPU orchestration validation for the Jonathan MLOps/LLMOps platform.

Requested SKU: Standard_ND96asr_v4
Alternative SKU: Standard_ND96amsr_A100_v4
Requested quantity: 256 A100 GPUs
Required VM quantity: 32 VMs
Required vCPU quantity: 3,072 vCPU
Requested regions: East US, North Central US, South Central US, West US 2, UK South
Requested duration: 1 week, maximum 1 month
Allocation flexibility: Split-region allocation acceptable
Business impact: $600K PO confirmed, GPU capacity is the main blocker
```

### Customer Scenario & Desired Outcome

```text
Acryl AI is planning to run large-scale distributed multi-GPU orchestration validation for the Jonathan MLOps/LLMOps platform on Azure.

The desired outcome is to secure deployable InfiniBand-enabled A100 GPU capacity on Azure so the customer can proceed with the validation workload.
```

### Customer Impact

```text
The customer has a confirmed $600K PO, and GPU capacity is the main blocker.

If Microsoft cannot provide feasible A100 GPU capacity, the Azure workload execution and related business opportunity are at risk.
```

### Workaround Details

```text
No confirmed Azure workaround is available yet.

Split-region allocation is acceptable if the requested 256 A100 GPUs cannot be fulfilled in a single region.
```

### Additional Comments

```text
The key customer requirement is InfiniBand-enabled GPU capacity.

The requested A100 capacity is 256 GPUs using Standard_ND96asr_v4 as the primary SKU. Standard_ND96amsr_A100_v4 can be considered as an alternative SKU.

Product Availability by Region shows possible A100 candidate regions for Standard_ND96asr_v4 including East US, North Central US, South Central US, West US 2, and UK South.

The customer can accept split-region allocation if capacity cannot be provided in a single region.
```
